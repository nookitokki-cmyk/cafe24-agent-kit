# 카트 JS 재구축 — 거래 페이지 완전 커스텀 (세 번째 트랙)

> **[프로토타입 · 검증됨 2026-07-06 skin2 라이브]** 장바구니·주문서처럼 **표를 뜯으면 주문이 깨지는** 거래 페이지를, 표를 안 뜯고 **100% 커스텀 디자인**으로 만드는 방법. 프리미엄 카페24 테마들이 실제 쓰는 방식.
> 재사용 코드: `snippets/_nk-cart-app/`

---

## 0. 세 트랙 중 위치

| 바꿀 것 | 트랙 |
|---|---|
| 색·톤만 | 오버라이드 (토대정리 + CSS) |
| 구조(표시 페이지: 목록·상세·홈) | **재마크업** (`module-remarkup.md`) |
| **거래 페이지(카트·주문서) 완전 커스텀** | **JS 재구축 (이 문서)** |

→ 거래 표는 재마크업(de-table)하면 주문이 깨진다(`module-remarkup.md` §5, `evidence-labels.md` B-2). 그래서 표를 **숨기고 그 위에 새 UI를 얹는다.**

---

## 1. 원리 (4단계)

```
① 카페24 원본 카트(표·주문함수)를 화면 밖으로 숨긴다 (기능은 살아있음)
② JS가 그 안에서 상품명·가격·수량·총액을 읽는다
③ 내 맘대로 만든 커스텀 UI(카드·요약·버튼)를 그린다
④ 내 버튼 = 숨긴 원본의 진짜 버튼을 대신 눌러준다 (프록시 클릭)
```

- 표를 **안 뜯으므로** rowspan·배송그룹·엔진 주입 폼이 전부 살아있어 **주문이 안 깨진다.**
- 겉모습은 내가 만든 DOM이라 **픽셀 단위 완전 커스텀.**

---

## 2. 안전 규칙 (오늘 삽질로 확정 — 안 지키면 깨짐)

1. **숨김은 `position:absolute; left:-99999px` 로만.** `visibility:hidden`/`display:none` 은 **금지** — 숨긴 요소는 `.click()`이 안 먹혀 **주문·수량·삭제가 작동 안 한다.** (오늘 이걸로 주문 실패 → 화면 밖 배치로 고침)
2. **주문 링크는 텍스트로 정확히 선택.** `a[link-order]` 첫 번째를 잡으면 **"스토어픽업 상품 선택"** 같은 엉뚱한 링크가 걸린다. `"전체상품주문"` 텍스트로 필터할 것.
3. **버튼은 프록시.** 커스텀 +/−·삭제·주문은 카페24 함수를 재구현하지 말고 **숨긴 원본 `<a>`를 `.click()`** 한다. 카페24가 페이지를 리로드하면 스크립트가 다시 그린다.
4. **엔진 초기화 후 실행.** `window.load` + `setTimeout` 으로 카페24 카트 JS가 폼을 주입한 뒤 읽는다.


## 4. 현재 상태 (정직)

- **[검증됨]**: 단순 상품 1건 렌더 + `전체상품주문` → 주문서 라우팅. 수량 +/−·삭제 배선 연결.
- **[검증필요 / 하드닝]**: 옵션 상품 · 품절 · 배송그룹 여러 개 · 수량 실시간 동기화 정밀도 · 다중 상품 레이아웃. → 실무 적용 전 해당 케이스를 연습 스킨에서 실증할 것.
