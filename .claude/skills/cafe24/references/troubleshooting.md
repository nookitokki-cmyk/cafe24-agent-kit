## 카테고리 1: 화면에 아무것도 안 나옴

> **⚠️ [모듈 표기 기준 · 2026-07-06 정정 완료]** 카페24에서 모듈은 **`module="모듈명"` HTML 속성**으로 씁니다 — 예: `<div module="product_list">...</div>`. 진열 개수 등 설정은 그 module 요소 **안에** `<!-- $count = 4 -->` 주석으로 넣습니다. **`<!--@module()-->` / `<!--@/module-->` 는 실제 지시어가 아닙니다**(과거 이 문서 예시가 이 가짜 표기를 썼으나 전부 실제 속성 형태로 교정함). 실제 지시어는 `@layout`·`@contents`·`@css`·`@js`·`@import` **괄호 단일태그**뿐입니다. (SKILL.md §1)

---

### 1-1. 모듈을 넣었는데 내용이 빈칸

**증상:** `<div module="...">` 를 넣었는데 해당 영역이 완전히 비어 있거나 아무 내용도 표시되지 않음

**원인:**
- `module=""` 속성에 오타가 있음 (대소문자 포함)
- 현재 페이지에서 지원하지 않는 모듈을 사용함 (예: 상품 목록 모듈을 메인 페이지가 아닌 곳에 넣음)
- 해당 모듈 파일(`/module/모듈명.html`)이 SFTP에 없음

**확인법:**
1. 카페24 어드민 > 디자인 > 스마트디자인 편집창 좌측 "모듈 목록"에서 모듈 ID를 복사 (직접 입력 금지)
2. 모듈 ID에 공백, 대소문자 오류가 없는지 비교
3. 카페24 공식 문서에서 해당 페이지에 사용 가능한 모듈 목록 확인

**수정법:**
```html
<!-- 잘못된 예 (대소문자 오류: L이 대문자) -->
<div module="product_List"> ... </div>

<!-- 올바른 예 (전부 소문자) -->
<div module="product_list"> ... </div>
```

**예방법:** 모듈 ID는 반드시 편집창의 모듈 목록에서 클릭해서 삽입. 직접 입력하면 오타 위험이 높음

---

### 1-2. 상품이 하나도 안 보임

**증상:** 상품 목록 모듈이 있는데 상품이 0개로 표시되거나 빈 목록만 나옴

**원인:**
- 해당 카테고리에 상품이 배치되어 있지 않음
- `display_count=0` 또는 옵션 값이 0으로 설정됨
- 모듈 옵션의 카테고리 번호가 실제 카테고리 번호와 다름

**확인법:**
1. 카페24 어드민 > 상품 관리 > 카테고리 관리에서 해당 카테고리 클릭
2. 카테고리 내 상품 배치 여부 확인
3. 스킨 HTML에서 `display_count` 값 확인 (0이면 안 나옴)

**수정법:**
```html
<!-- 잘못된 예: 진열 개수가 0 (module 요소 안 $count 주석) -->
<div module="product_list_main">
  <!-- $count = 0 -->
</div>

<!-- 올바른 예: 최소 1 이상 -->
<div module="product_list_main">
  <!-- $count = 4 -->
</div>
```

**예방법:** 상품 목록 모듈 삽입 후 반드시 어드민에서 카테고리 배치 상태를 먼저 확인

---

### 1-3. 이미지가 깨짐 (X 표시 또는 빈 공간)

**증상:** 이미지 자리에 X 표시가 나오거나 빈 박스만 보임

**원인:**
- 이미지 경로 변수 오타 (예: `{$product_img1}` → `{$product_image_1}`)
- 해당 모듈 스코프 밖에서 모듈 전용 이미지 변수를 사용함
- SFTP에 이미지 파일을 업로드하지 않음
- 이미지 경로가 절대경로가 아닌 상대경로로 잘못 작성됨

**확인법:**
1. 변수명을 카페24 공식 변수 목록과 대조
2. 해당 이미지 변수가 올바른 모듈 안에 있는지 확인
3. SFTP 클라이언트에서 이미지 파일 존재 여부 확인

**수정법:**
```html
<!-- 잘못된 예: 변수명 오타 -->
<img src="{$product_img_1}" alt="상품 이미지">

<!-- 올바른 예 -->
<img src="{$product_image_1}" alt="{$product_name}">

<!-- 직접 업로드 이미지 경로 예시 -->
<img src="//쇼핑몰도메인.com/web/upload/banner/main_banner.jpg" alt="메인 배너">
```

**예방법:** 이미지 변수명은 반드시 `~/.claude/skills/cafe24/references/variables.md` 파일 참조

## 카테고리 2: 변수가 값으로 안 바뀜 (빈 값 또는 오타 시 글자 그대로)

---

### 2-1. `{$product_name}` 이 값으로 안 바뀜 (빈 값·오타 시만 글자 그대로)

**증상:** 상품명 자리가 **비어 있음**(모듈 밖 사용 시) 또는 `{$product_name}` 텍스트가 그대로 보임(오타 시)

**원인:**
- **모듈 범위(스코프) 밖에서 사용 → 빈 값** ([실측 2026-07-06]. 문자 그대로 노출 아님)
- 해당 변수가 현재 페이지 타입에서 지원되지 않음 → 빈 값
- **변수명 오타(중괄호·$ 누락) → 이 경우에만 `{$...}` 문자가 그대로 보임**

**확인법:**
1. 해당 변수가 올바른 모듈 요소(`module="모듈명"` 속성) 안에 있는지 확인
2. 현재 페이지(메인/상품상세/목록 등)에서 해당 변수가 지원되는지 공식 문서 확인
3. `{$` 와 `}` 가 제대로 감싸져 있는지 확인

**수정법:**
```html
<!-- 잘못된 예: 모듈 밖에서 상품명 변수 사용 -->
<div class="nk-title">{$product_name}</div>
<div module="product_detail"> 내용 </div>

<!-- 올바른 예: module="..." 속성 요소 안에서 사용 -->
<div module="product_detail">
  <div class="nk-title">{$product_name}</div>
</div>
```

**예방법:** 변수를 쓰기 전에 "이 변수가 어떤 모듈에 속하는지" 먼저 확인

---

### 2-2. 가격이 숫자로 안 나옴 (`{$price}` 등 틀린 변수명)

**증상:** 가격 자리가 **빈 값**(숫자가 안 나옴). 단, 중괄호·`$` 누락 등 **표기 오타**일 때만 `{$price}` 글자가 그대로 보임 ([실측 2026-07-06]. 문자 그대로 노출 아님)

**원인:**
- 카페24 가격 변수명은 `{$price}`가 아닌 맥락마다 다름 → 틀린 이름은 **빈 값**
- 모듈 스코프 문제 (상품 목록 모듈 밖에서 사용) → **빈 값**
- 해당 페이지 타입에서 지원하지 않는 변수 사용 → **빈 값**

**확인법:**
올바른 가격 변수명 (상품 모듈 안에서만 유효):
- 판매가(할인 적용가): `{$product_sale_price}`
- 정가(상품가): `{$product_price}`
- (천 단위 콤마는 모디파이어로: `{$product_price|number_format}`)

**수정법:**
```html
<!-- 올바른 사용 예 (상품 목록 모듈 내부, 올바른 가격 변수) -->
<div module="product_list">
  <span class="nk-price">{$product_price|number_format}원</span>
</div>
<!-- 주의: {$price}는 존재하지 않는 변수 → 빈 값. 정가는 {$product_price} -->
```

**예방법:** 가격 변수 사용 전 반드시 해당 모듈 스코프 안에 있는지 확인

## 카테고리 3: 스타일이 안 먹힘

---

### 3-1. CSS를 적용했는데 화면에 반영이 안 됨

**증상:** `style.css`에 색상이나 폰트를 변경했는데 쇼핑몰 화면에서 바뀌지 않음

**원인:**
- EZ 에디터가 자동 생성한 인라인 `style=""` 속성이 CSS를 덮어씀 (인라인 스타일은 외부 CSS보다 항상 우선함)
- 브라우저 캐시가 남아 있음
- SFTP 업로드를 하지 않아 서버 파일이 구버전

**확인법:**
1. 크롬 개발자 도구(F12) > 요소 탭에서 해당 요소 클릭
2. 오른쪽 스타일 패널에서 내가 적용한 CSS가 취소선으로 표시되는지 확인 (취소선 = 다른 스타일에 덮임)
3. 인라인 `style=""` 이 있는지 확인

**수정법:**
```css
/* EZ 에디터 인라인 스타일 오버라이드 시에만 !important 허용 */
/* 반드시 범위를 좁게 지정해서 다른 요소에 영향 안 주게 */
.xans-product-detail .nk-price {
  color: #333 !important;
}

/* 일반 CSS는 !important 없이 작성 */
.nk-section-title {
  font-size: 24px;
  font-weight: 700;
}
```

**예방법:** EZ 에디터에서 색상/폰트를 바꿀 때는 EZ 설정 패널을 사용. 커스텀 CSS와 EZ 설정은 영역을 분리해서 관리

---

### 3-2. 모바일에서만 레이아웃이 깨짐

**증상:** PC에서는 정상 표시되는데 스마트폰에서 보면 줄이 겹치거나 이미지가 넘침

**원인:**
- 모바일용 미디어쿼리가 없음
- `px` 고정값을 사용해서 작은 화면에서 넘침 (overflow)
- `vw`, `%` 같은 유연한 단위를 사용하지 않음

**확인법:**
크롬 개발자 도구(F12) > 좌상단 스마트폰 아이콘 클릭 > 화면 너비를 375px로 설정 후 확인

**수정법:**
```css
/* PC 기본 스타일 */
.nk-banner {
  width: 1200px;
  display: flex;
  gap: 20px;
}

/* 768px 이하 태블릿/모바일 */
@media (max-width: 768px) {
  .nk-banner {
    width: 100%;       /* 고정 px 대신 % 사용 */
    flex-direction: column;  /* 가로 → 세로 정렬 */
    gap: 12px;
  }
}

/* 375px 이하 소형 스마트폰 */
@media (max-width: 375px) {
  .nk-banner {
    gap: 8px;
  }
}
```

**예방법:** 처음부터 모바일 먼저 설계(Mobile First)하고 PC 확장 방식으로 작성

---

### 3-3. 수정사항이 저장 후에도 반영이 안 됨

**증상:** 분명히 저장 버튼을 눌렀는데 새로고침해도 이전 화면 그대로

**원인:**
1. SFTP 파일 업로드를 깜빡함 (스마트디자인 HTML 방식에서 자주 발생)
2. 브라우저 캐시가 예전 파일을 보여주고 있음
3. CDN 캐시 (카페24 서버 측 캐시가 아직 갱신 안 됨)

**확인법:**
1. SFTP 클라이언트에서 파일 업로드 시각 확인
2. 강력 새로고침으로 캐시 무시: **Ctrl + Shift + R** (윈도우) / **Cmd + Shift + R** (맥)
3. 시크릿 창(크롬: Ctrl+Shift+N)에서 열어보기

**수정법:**
```
1단계: SFTP 클라이언트(FileZilla 등)에서 수정한 파일 재업로드
2단계: 브라우저에서 Ctrl+Shift+R 강력 새로고침
3단계: 그래도 안 되면 5~10분 대기 후 재확인 (CDN 캐시 반영 시간)
```

**예방법:** 수정 → 저장 → SFTP 업로드를 한 세트로 습관화. 체크리스트 사용 권장

## 카테고리 4: EZ 에디터 충돌

---

### 4-1. EZ 에디터에서 바꿨는데 엉뚱한 곳이 바뀜

**증상:** EZ 에디터에서 특정 텍스트 색상을 바꿨는데 다른 영역 텍스트까지 같이 바뀜

**원인:**
- EZ 에디터가 스타일을 적용할 때 클래스 단위로 적용하므로 같은 클래스를 가진 모든 요소에 영향
- EZ가 자동으로 추가한 클래스명이 다른 요소와 겹침

**확인법:**
1. 크롬 개발자 도구 > 해당 요소의 클래스명 확인
2. 페이지 내 같은 클래스명을 가진 다른 요소가 있는지 확인

**수정법:**
```css
/* EZ 에디터가 생성한 클래스명에 nk- 접두사 클래스로 덮어씀 */
/* 범위를 최대한 좁혀서 특정 영역에만 적용 */
.xans-board-list .nk-board-title {
  color: #222;
}

/* 다른 영역의 같은 클래스에는 영향 안 가도록 부모 선택자 활용 */
.xans-product-detail .nk-price {
  color: #e04040;
}
```

**예방법:** EZ 에디터에서 전체 적용이 아닌 섹션/블록 단위로 스타일 변경. 커스텀 CSS 작성 시 항상 `.xans-*` 부모 선택자로 범위 제한

---

### 4-2. EZ 에디터 저장 후 내 코드가 사라짐

**증상:** 직접 HTML을 수정했는데 EZ 에디터에서 저장하면 코드가 원래대로 돌아오거나 삭제됨

**원인:**
- EZ 에디터는 저장 시 EZ 블록 구조를 기준으로 HTML을 **재생성**함
- EZ 블록 바깥에 직접 쓴 커스텀 코드는 재생성 과정에서 덮어씌워짐

**확인법:**
EZ 편집 화면에서 "HTML 직접 편집" 버튼으로 들어갔을 때 수정한 영역이 EZ 블록 안에 있는지 확인

**수정법:**
```
방법 1: EZ가 건드리지 않는 영역에 코드 배치
- 헤더/푸터는 EZ 블록 바깥에 위치 → 안전
- EZ 블록 사이 공백 영역에 HTML 삽입 가능

방법 2: CSS만 별도 파일로 분리
- /resource/style.css 에 커스텀 CSS 작성
- EZ 저장이 CSS 파일을 덮어쓰지 않음

방법 3: EZ "코드 삽입" 위젯 사용
- EZ 에디터 > + 블록 추가 > 코드 삽입
- 이 안에 넣은 HTML은 EZ가 재생성해도 유지됨
```

**예방법:** EZ와 HTML 방식을 혼용할 때는 역할을 명확히 분리. EZ = 레이아웃/콘텐츠, 커스텀 파일 = 기능/스타일

---

### 4-3. EZ 에디터에서 블록을 삭제했는데 빈 공간이 남음

**증상:** EZ 에디터에서 섹션을 삭제했는데 그 자리에 빈 여백이 그대로 남음

**원인:**
- 삭제한 블록의 CSS(`margin`, `padding`)가 다른 파일에 남아 있음
- EZ 블록이 아닌 커스텀 HTML/CSS에 해당 영역이 정의되어 있음

**확인법:**
크롬 개발자 도구 > 빈 영역 클릭 > 어떤 요소가 그 공간을 차지하는지 확인

**수정법:**
```css
/* 크롬 개발자 도구에서 찾은 요소의 클래스명으로 여백 제거 */
.nk-empty-section {
  display: none;
  /* 또는 */
  margin: 0;
  padding: 0;
  height: 0;
}
```

**예방법:** EZ 블록 삭제 후 반드시 크롬 개발자 도구로 실제 DOM에 남은 요소가 있는지 확인

## 카테고리 5: 게시판 관련

### 5-1. 게시판 모듈 넣었는데 "게시판 번호가 다릅니다" 오류

**증상:** 게시판 영역에 오류 메시지가 표시되거나 게시판이 불러와지지 않음

**원인:**
- 모듈에 지정한 게시판 번호와 어드민에서 실제 게시판 번호가 다름
- `board_list_{no}` 에서 `{no}` 를 잘못 입력함

**확인법:**
1. 카페24 어드민 > 게시판 > 게시판 관리 클릭
2. 대상 게시판의 "게시판 번호"(보통 숫자) 확인
3. 스킨 HTML의 모듈 코드에서 번호 대조

**수정법:**
```html
<!-- 어드민에서 게시판 번호가 3번이라면 -->

<!-- 잘못된 예 (1번 게시판 모듈) -->
<div module="board_list_1"> ... </div>

<!-- 올바른 예 (3번 게시판 모듈) -->
<div module="board_list_3"> ... </div>
```

**예방법:** 게시판 모듈 삽입 시 어드민에서 번호를 복사해서 붙여넣기

---

### 5-2. 게시판 글 목록이 안 나오고 "등록된 게시물이 없습니다" 표시

**증상:** 게시판 모듈은 정상 로드되는데 내용이 없는 상태로 표시됨

**원인:**
- 실제로 게시글이 없음
- 게시판 설정에서 "비밀글 여부", "권한 설정"이 잘못되어 방문자에게 안 보임

**확인법:**
1. 어드민 > 게시판에 실제 게시글이 있는지 확인
2. 게시판 설정 > 글 읽기 권한이 "모두"로 설정되어 있는지 확인

**수정법:**
```
어드민 > 게시판 > [해당 게시판] > 기본설정 > 읽기 권한 > "모두" 선택 후 저장
```

**예방법:** 게시판 신규 생성 후 권한 설정을 먼저 확인하고 테스트 글 1개 작성해서 확인

## 카테고리 6: 모바일/반응형

---

### 6-1. PC에서는 괜찮은데 모바일에서 이미지가 넘침

**증상:** 스마트폰에서 볼 때 이미지가 화면 밖으로 삐져나오거나 가로 스크롤이 생김

**원인:**
- 이미지에 고정 `width` 값(px)이 설정되어 있음
- `max-width` 처리가 안 되어 있음

**확인법:**
크롬 개발자 도구 > 모바일 뷰 > 이미지 클릭 > `width` 스타일 값 확인

**수정법:**
```css
/* 모든 이미지에 기본 반응형 처리 */
.nk-section img {
  max-width: 100%;
  height: auto;
  display: block;
}

/* 특정 이미지 너비 제한 필요 시 */
.nk-banner-img {
  width: 100%;
  max-width: 600px;
  aspect-ratio: 16 / 9;  /* 비율 고정 */
  object-fit: cover;
}
```

**예방법:** 이미지 삽입 시 항상 `max-width: 100%`를 기본으로 적용

---

### 6-2. 터치 버튼이 너무 작아서 눌리지 않음

**증상:** 모바일에서 버튼이나 링크를 누르려고 해도 잘 눌리지 않거나 옆 버튼이 같이 눌림

**원인:**
- 터치 타겟 크기가 44px 미만 (모바일 접근성 최소 기준)
- 버튼끼리 간격이 너무 좁음

**확인법:**
크롬 개발자 도구 > 모바일 뷰 > 버튼 클릭 > `width`, `height` 값 확인 (44px 미만이면 위험)

**수정법:**
```css
/* 모바일에서 버튼/링크 터치 영역 확보 */
@media (max-width: 768px) {
  .nk-btn {
    min-height: 44px;   /* 최소 높이 44px */
    min-width: 44px;    /* 최소 너비 44px */
    padding: 12px 20px; /* 내부 여백으로 크기 확보 */
    display: inline-flex;
    align-items: center;
    justify-content: center;
  }

  /* 버튼 사이 간격도 최소 8px 이상 */
  .nk-btn + .nk-btn {
    margin-left: 8px;
  }
}
```

**예방법:** 버튼 디자인 시 모바일 크기를 처음부터 44px 이상으로 설계

---

### 6-3. 모바일에서 가로 스크롤이 생김

**증상:** 스마트폰에서 페이지를 옆으로 스크롤할 수 있음 (의도하지 않은 가로 스크롤)

**원인:**
- 화면보다 넓은 요소가 존재 (고정 px 너비, 또는 음수 margin)
- `overflow: hidden` 처리가 안 되어 있음

**확인법:**
크롬 개발자 도구 > 모바일 뷰 > 화면보다 넓은 요소 찾기 (형광 표시되는 요소 확인)

**수정법:**
```css
/* 전체 가로 스크롤 방지 */
body {
  overflow-x: hidden;
}

/* 래퍼도 함께 처리 */
.nk-wrapper {
  max-width: 100vw;
  overflow-x: hidden;
}

/* 모바일에서 특정 요소 너비 제한 */
@media (max-width: 768px) {
  .nk-table {
    width: 100%;
    overflow-x: auto;   /* 표 같은 건 스크롤 허용 */
    display: block;
  }
}
```

**예방법:** 페이지 제작 완료 후 반드시 모바일 뷰에서 가로 스크롤 여부 최종 확인

### 7-1. SFTP 연결이 안 됨

**증상:** FileZilla 등 SFTP 클라이언트에서 "연결 실패", "Host를 찾을 수 없음" 오류

**원인:**
- 연결 정보(호스트/포트/계정) 오류
- 카페24 서버 SFTP 접근 권한이 비활성화
- 포트 번호를 22로 잘못 입력 (카페24는 기본 포트 다름)

**확인법:**
카페24 어드민 > 디자인 > 파일 관리 > FTP/SFTP 정보 확인

```
카페24 SFTP 기본 정보
- 호스트: 쇼핑몰도메인.cafe24.com
- 포트: 22 (변경 가능, 어드민에서 확인)
- 아이디: 카페24 아이디
- 비밀번호: 카페24 비밀번호
- 프로토콜: SFTP (FTP가 아닌 SFTP 선택)
```

**수정법:**
```
FileZilla 기준:
파일 > 사이트 관리자 > 새 사이트 > 
프로토콜: SFTP
호스트: [쇼핑몰ID].cafe24.com
포트: 어드민에서 확인한 번호
로그온 유형: 일반
사용자: 카페24 아이디
비밀번호: 카페24 비밀번호
```

**예방법:** SFTP 연결 정보를 처음에 저장해두고, 비밀번호 변경 시 SFTP도 함께 업데이트

### 7-2. 파일을 SFTP로 업로드했는데 이전 버전이 계속 나옴

**증상:** 분명히 수정 파일을 업로드했는데 쇼핑몰에서는 여전히 예전 파일로 보임

**원인:**
- 카페24 CDN(콘텐츠 전송 네트워크)에서 이전 파일을 캐싱
- 브라우저 캐시가 남아 있음
- CSS, JS 파일은 캐시 유지 시간이 길어 더 오래 걸릴 수 있음

**확인법:**
1. 시크릿 창에서 열어보기 (브라우저 캐시 무시)
2. 크롬 개발자 도구 > 네트워크 탭 > "캐시 사용 안 함" 체크 후 새로고침
3. URL 뒤에 `?v=2` 같은 쿼리 파라미터 추가해서 강제 새로고침

**수정법:**
```
즉시 해결: Ctrl+Shift+R (강력 새로고침)

CSS/JS 파일 캐시 갱신 방법:
HTML에서 파일 경로에 버전 쿼리 추가
```
```html
<!-- 수정 전 -->
<link rel="stylesheet" href="/resource/style.css">

<!-- 수정 후 (버전 번호 올릴 때마다 캐시 갱신) -->
<link rel="stylesheet" href="/resource/style.css?v=202601211">
```

**예방법:** CSS/JS 파일에 버전 쿼리를 붙이는 습관 들이기. 날짜 기반 버전 번호 추천 (YYYYMMDD)

---

### 7-3. 스킨 파일이 두 개 있는데 어떤 파일을 수정해야 하는지 모름

**증상:** SFTP로 접속했을 때 비슷한 이름의 폴더나 파일이 여러 개 있어 어느 것을 수정해야 하는지 혼란

**원인:**
- 카페24는 PC 스킨과 모바일 스킨이 분리되어 있는 경우가 있음
- 다국어 대응 시 폴더가 나뉘기도 함
- 이전 버전 스킨과 현재 스킨이 함께 있을 수 있음

**확인법:**
```
카페24 SFTP 폴더 구조:
/
├── skin/             ← PC 스킨 폴더
│   └── shop/
│       └── [스킨명]/  ← 현재 사용 중인 스킨 폴더 (어드민 > 디자인 > 내 디자인에서 확인)
├── mobile/           ← 모바일 전용 스킨 (구버전)
└── web/upload/       ← 업로드 이미지 폴더 (수정 불필요)
```

**확인 순서:**
1. 카페24 어드민 > 디자인 > 내 디자인 > 현재 사용 중인 스킨 이름 확인
2. SFTP에서 `/skin/shop/[스킨명]/` 폴더 접속
3. 반응형 스킨이면 PC/모바일 폴더가 하나로 통합되어 있음

**수정법:**
```
반응형 스킨 사용 중 (권장):
→ /skin/shop/[스킨명]/ 폴더의 파일만 수정

구형 PC+모바일 분리 스킨 사용 중:
→ PC: /skin/shop/[스킨명]/
→ 모바일: /skin/shop_m/[스킨명]/ (또는 /mobile/)
→ 두 폴더 모두 수정 필요
```

**예방법:** 프로젝트 시작 전 `CLAUDE.md` 또는 `05_work-log.md`에 스킨 폴더 경로를 기록해두기
