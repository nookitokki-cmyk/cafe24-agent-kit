# 카페24 스마트디자인 모디파이어 & 문법 사전

> 카페24 스마트디자인(구매템플릿/NK 템플릿 엔진)에서 사용하는 모든 `|modifier` 문법과 특수 케이스를 정리한 참고서.
> 비코더도 바로 복사·붙여넣기로 활용할 수 있도록 한국어 설명 + 실제 예시를 모든 항목에 포함.
> 출처: kimyoungwopo/cafe24-smart-design (MIT) + 카페24 공식문서 sdsupport.cafe24.com

---

## 모디파이어 기본 규칙

- 변수 뒤에 파이프(`|`)를 붙이고 모디파이어 이름을 씀: `{$변수명|모디파이어}`
- 모디파이어는 **체이닝(연결)** 가능: `{$변수명|모디파이어1|모디파이어2}`
- 콜론(`:`)으로 옵션을 전달: `{$변수명|모디파이어:옵션값}`
- 변수가 비어 있으면 대부분의 모디파이어는 빈 문자열을 그대로 반환

## 1. 출력 모디파이어

### `|cut:N`

**기능:** 문자열을 N자로 잘라내고, 잘린 경우 자동으로 `...`을 붙임

**문법:**
```
{$상품명|cut:20}
```

**예시:**
- Before (원본 값): `우아한 실크 블라우스 봄 신상품 특가`
- After (렌더링 결과): `우아한 실크 블라우스 봄 신...`

**주의:** 한글 1글자 = 1자로 계산. 영문/숫자도 1자씩 계산.

> **⚠️ `{$product_name}`엔 `|cut` 쓰지 말 것** `[실측 2026-07-06]`: 카페24가 `{$product_name}`을 `<span style="...">이름</span>`으로 **감싸서** 출력한다. 여기에 `|cut:2`를 걸면 텍스트가 아니라 **`<span` 마크업이 잘려** `<s...`처럼 깨진다. → 목록 카드 이름 줄임은 `|cut` 대신 **CSS `text-overflow:ellipsis`** 로 처리. (`|cut`은 순수 텍스트 변수에만)

---

### `|cut:N:suffix`

**기능:** 잘릴 때 붙는 후미 텍스트를 직접 지정. `...` 대신 원하는 문자를 사용

**문법:**
```
{$상품명|cut:20:...}
{$상품명|cut:15:더보기}
```

**예시:**
- Before: `가을 겨울 신상 롱코트 브라운 컬러`
- After (`|cut:10:..`): `가을 겨울 신상..`

**주의:** suffix 자체의 길이는 N에 포함되지 않는다. suffix에 HTML 태그는 사용하지 말 것.

---

### `|nl2br`

**기능:** 텍스트 안의 줄바꿈 문자(`\n`)를 HTML `<br>` 태그로 변환. 관리자 페이지에서 여러 줄로 입력한 내용을 화면에서도 줄바꿈으로 표시할 때 사용

**문법:**
```
{$상품요약설명|nl2br}
```

**예시:**
- Before (원본 값): `소재: 면 100%\n사이즈: S/M/L\n원산지: 국내`
- After (HTML 출력): `소재: 면 100%<br>사이즈: S/M/L<br>원산지: 국내`

**주의:** 이미 `<br>`이 포함된 HTML 콘텐츠에 적용하면 줄바꿈이 두 번 들어갈 수 있으므로 중복 적용 금지.

---

### `|escape`

**기능:** HTML 특수문자를 안전한 코드로 변환. 사용자가 입력한 텍스트를 그대로 출력할 때 HTML이 깨지거나 스크립트가 실행되는 것을 방지

| 원본 문자 | 변환 결과 |
|-----------|-----------|
| `<` | `&lt;` |
| `>` | `&gt;` |
| `&` | `&amp;` |
| `"` | `&quot;` |
| `'` | `&#039;` |

**문법:**
```
{$검색어|escape}
```

**예시:**
- Before: `<script>alert('해킹')</script>`
- After: `&lt;script&gt;alert(&#039;해킹&#039;)&lt;/script&gt;`

**주의:** 보안상 사용자 입력값은 반드시 `|escape` 처리. `|noescape`와 절대 혼용 금지.

---

### `|noescape`

**기능:** HTML 태그를 변환하지 않고 원본 그대로 출력. 관리자가 직접 입력한 HTML 콘텐츠(상품 상세설명 등)를 화면에 렌더링할 때 사용

**문법:**
```
{$상품상세설명|noescape}
```

**예시:**
- Before: `<p><strong>특가</strong> 오늘만 30% 할인</p>`
- After (렌더링): **특가** 오늘만 30% 할인 (태그가 실제 HTML로 동작)

**주의:** 사용자 입력값에 절대 사용 금지 (XSS 공격 위험). 관리자 직접 입력 영역에만 한정 사용.

---

### `|upper`

**기능:** 영문 소문자를 대문자로 변환

**문법:**
```
{$브랜드영문명|upper}
```

**예시:**
- Before: `hera beauty`
- After: `HERA BEAUTY`

**주의:** 한글에는 효과 없음.

---

### `|lower`

**기능:** 영문 대문자를 소문자로 변환

**문법:**
```
{$카테고리코드|lower}
```

**예시:**
- Before: `OUTER-JACKET`
- After: `outer-jacket`

---

### `|number_format`

**기능:** 숫자에 천 단위 콤마(`,`)를 자동으로 삽입. 가격, 재고 수량 등 숫자 표시에 필수

**문법:**
```
{$상품가격|number_format}
```

**예시:**
- Before: `129000`
- After: `129,000`

**주의:** 소수점 처리는 별도 옵션 없음. 소수점이 있는 값은 반올림 처리됨.

---

### `|strip_tags`

**기능:** 문자열에 포함된 모든 HTML 태그를 제거하고 텍스트만 추출. SEO 메타 디스크립션 생성이나 알림 문자 발송 내용 정제에 유용

**문법:**
```
{$상품상세설명|strip_tags}
```

**예시:**
- Before: `<p><strong>베스트셀러</strong> 아이템! <br>한정 수량</p>`
- After: `베스트셀러 아이템! 한정 수량`

## 2. 조건 모디파이어

### `|display`

**기능:** 변수 값이 존재하면 요소를 표시, 비어 있으면 `style="display:none"` 속성을 자동 추가해 숨김. 값이 있는 경우의 기본 display 타입은 `block`

**문법:**
```
<div {$배송비|display}>무료배송</div>
```

**예시:**
- Before (값이 있는 경우): `<div>무료배송</div>` (정상 표시)
- Before (값이 없는 경우): `<div style="display:none">무료배송</div>` (숨김)

**주의:** 속성 자리에 직접 작성. `{$변수|display}` 자체가 `style="display:none"` 또는 빈 문자열로 렌더링됨.

---

### `|display:block`

**기능:** 값이 있으면 `display:block`으로 명시 표시. 인라인 요소를 블록으로 강제 처리할 때

**문법:**
```
<span {$이벤트뱃지|display:block}>SALE</span>
```

---

### `|display:inline`

**기능:** 값이 있으면 `display:inline`으로 표시. 블록 요소를 인라인으로 처리할 때

**문법:**
```
<div {$할인율|display:inline}>{$할인율}% 할인</div>
```

---

### `|checked`

**기능:** 체크박스나 라디오 버튼 input 태그에 `checked` 속성을 자동으로 추가. 값이 참(`true`, `1`, `Y` 등)이면 checked, 아니면 아무것도 출력 안 함

**문법:**
```
<input type="checkbox" {$수신동의|checked}> 이메일 수신 동의
```

**예시:**
- 값이 Y일 때: `<input type="checkbox" checked> 이메일 수신 동의`
- 값이 N일 때: `<input type="checkbox"> 이메일 수신 동의`

---

### `|selected`

**기능:** select 드롭다운의 option 태그에 `selected` 속성을 자동 추가. 현재 선택된 옵션 표시에 사용

**문법:**
```
<option value="KR" {$국가코드|selected}>대한민국</option>
```

**예시:**
- `$국가코드` 값이 `KR`일 때: `<option value="KR" selected>대한민국</option>`
- 다른 값일 때: `<option value="KR">대한민국</option>`

**주의:** `selected` 모디파이어의 값 자동 비교 동작 여부는 **[검증필요]**. (`<!--[if]-->` 조건문은 **미작동**이므로 함께 쓰지 말 것 — [실측-반증 2026-07-06])

---

### `|disabled`

**기능:** 폼 요소(input, button, select 등)에 `disabled` 속성을 추가. 값이 참이면 비활성화

**문법:**
```
<button {$품절여부|disabled}>구매하기</button>
```

**예시:**
- 품절일 때: `<button disabled>구매하기</button>`
- 재고 있을 때: `<button>구매하기</button>`

---

## 3. 이미지/URL 모디파이어

### `|resize:W:H`

**기능:** 이미지 URL에 리사이즈 파라미터를 자동으로 붙여서 지정한 크기의 이미지를 요청. 카페24 이미지 서버가 자동으로 크기를 조정해 응답

**문법:**
```
<img src="{$상품이미지|resize:200:200}" alt="{$상품명}">
<img src="{$상품이미지|resize:400:0}" alt="{$상품명}">  <!--높이 0 = 비율 자동-->
```

**예시:**
- Before (원본 URL): `https://shop-cdn.cafe24.com/data/goods/24/04/sample.jpg`
- After: `https://shop-cdn.cafe24.com/data/goods/24/04/sample.jpg?width=200&height=200`

**주의:** 카페24 CDN 이미지에만 동작. 외부 URL에는 효과 없음. 원본보다 크게 늘리면 화질이 저하됨.

---

### `|thumbnail`

**기능:** 이미지를 썸네일 크기(관리자에서 설정한 기본 썸네일 크기)로 변환하는 간단 버전. `|resize`보다 설정이 간단하나 크기 제어는 불가

**문법:**
```
<img src="{$상품이미지|thumbnail}" alt="{$상품명}">
```

---

### `|urlencode`

**기능:** 문자열을 URL에서 안전하게 사용할 수 있도록 인코딩. 검색어, 파라미터 값을 URL에 삽입할 때 한글이나 특수문자가 깨지는 것을 방지

**문법:**
```
<a href="/search?q={$검색어|urlencode}">검색 결과 보기</a>
```

**예시:**
- Before: `봄 신상 원피스`
- After: `%EB%B4%84+%EC%8B%A0%EC%83%81+%EC%9B%90%ED%94%BC%EC%8A%A4`

## 4. 날짜/숫자 모디파이어

### `|date_format:'포맷문자열'`

**기능:** 타임스탬프나 날짜 문자열을 원하는 형식으로 변환. PHP `date()` 함수와 동일한 포맷 코드 사용

**자주 쓰는 포맷 코드:**

| 코드 | 의미 | 예시 |
|------|------|------|
| `Y` | 4자리 연도 | 2026 |
| `m` | 2자리 월(01~12) | 06 |
| `d` | 2자리 일(01~31) | 21 |
| `H` | 24시간 시각(00~23) | 14 |
| `i` | 분(00~59) | 05 |
| `s` | 초(00~59) | 30 |
| `n` | 월(1~12, 앞 0 없음) | 6 |
| `j` | 일(1~31, 앞 0 없음) | 21 |

**문법 및 예시:**
```
{$주문일시|date_format:'Y-m-d'}
{$주문일시|date_format:'Y년 m월 d일'}
{$주문일시|date_format:'Y.m.d H:i'}
```

**렌더링 결과:**
- `2026-06-21`
- `2026년 06월 21일`
- `2026.06.21 14:05`

**주의:** 포맷 문자열은 반드시 작은따옴표로 감쌀 것. 큰따옴표 사용 시 오동작 가능.

---

### `|ceil`

**기능:** 소수점 이하를 올림하여 정수로 만듦

**문법:**
```
{$평균평점|ceil}
```

**예시:**
- Before: `4.2`
- After: `5`

---

### `|floor`

**기능:** 소수점 이하를 내림하여 정수로 만듦

**문법:**
```
{$배송예상일수|floor}
```

**예시:**
- Before: `3.9`
- After: `3`

---

### `|round`

**기능:** 소수점 이하를 반올림하여 정수로 만듦

**문법:**
```
{$할인율|round}
```

**예시:**
- Before: `33.5`
- After: `34`

---

### `|abs`

**기능:** 음수 값을 양수로 변환(절댓값). 음수로 저장된 할인 금액 등을 양수로 표시할 때

**문법:**
```
{$할인금액|abs}
```

**예시:**
- Before: `-5000`
- After: `5000`

## 5. 특수 문법 (카페24 전용 블록 문법)

### 조건 분기 — `|display` 필터로 (⚠️ `<!--[if]-->` 는 작동 안 함)

> **[실측-반증 · 2026-07-06 skin2 라이브]** `<!--[if 조건]-->...<!--[endif]-->` 주석 조건문은 **조건 분기가 작동하지 않는다.** 테스트에서 `<!--[if {$product_no} gt 9]-->큰값<!--[else]-->작은값<!--[endif]-->` 이 분기 없이 **양쪽(큰값·작은값)을 모두 출력**했다. → **사용 금지.** (기존 `eq/ne/gt/lt`·`else`·`and/or` 안내는 미작동이라 삭제)

**카페24의 실제 조건 분기 = `|display` 필터** (값 있으면 표시 / 비면 자동 `display:none`):

```html
<!-- 값이 있을 때만 노출. 비면 카페24가 자동으로 style="display:none" 삽입 -->
<span class="{$soldout_display|display}">품절</span>
<button class="{$buy_display|display}">구매하기</button>
```

- base 스킨도 이 방식을 씀(`{$xxx_display|display}` 형태가 곳곳에 있음).
- 이게 카페24의 표준 조건 노출 방법. 복잡한 로직만 JavaScript로 처리.

### 반복 진열 블록 내 인덱스: `{$key}`, `{$no}`

**기능:** 모듈 내 반복 진열 블록(li 자동반복)에서 현재 항목의 순번을 출력

| 변수 | 의미 |
|------|------|
| `{$key}` | 0부터 시작하는 인덱스 (0, 1, 2...) |
| `{$no}` | 1부터 시작하는 순번 (1, 2, 3...) |

**사용 위치:** 게시판 목록 등 일부 반복 블록. **[검증필요]**

> **⚠️ [실측 · 2026-07-06 skin2]** `product_listnormal`(상품목록) 반복 블록에서 `{$key}`·`{$no}`는 **빈 값**으로 나왔다. → **상품목록엔 쓰지 말 것.** 게시판 등 다른 모듈에서의 작동은 각 페이지 실측으로 확인하기 전까지 **[검증필요]**로 둔다.

**주의:** 모듈 바깥에서 쓰면 빈 값.

### 모듈 변수 스코프 규칙

**기능:** 카페24 스마트디자인 변수는 모듈 경계를 넘어 사용할 수 없음. 각 모듈은 독립적인 변수 공간을 가짐

**핵심 규칙:**
- 상품 목록 모듈 내 변수 → 상품 상세 모듈에서 참조 불가
- 장바구니 모듈 변수 → 헤더 영역에서 직접 출력 불가 (별도 전용 모듈 사용)
- 공통 값(쇼핑몰 이름, 회원 이름 등)도 전용 변수가 따로 있으나, 역시 해당 module scope 안에서만 치환됨

**공통 변수 (점 없는 형태만 사용 — module scope 안에서만 치환):**
```html
{$mall_name}       <!-- 쇼핑몰 이름 (variables.md 검증본) -->
```

> **⚠️ [실측-반증 · 2026-07-06]** `{$mall.mall_name}` 처럼 **점(.) 붙은 형태는 빈 값**으로 나왔다. kit 표준은 점 없는 `{$mall_name}`(variables.md·SKILL.md). **점 형태 사용 금지.**

### 지시어(directives)는 전부 `<!--@명령(경로)-->` 괄호 단일태그

> **⚠️ [정정 · 2026-07-06]** 아래 형태들은 **공식 근거가 없어 삭제**했다: `<!--@css-->...<!--/@css-->`(쌍태그 인라인 CSS), `<!--@html-->...<!--/@html-->`, `<!--@layout name="...">`·`<!--@section-->`·`<!--@placeholder-->`(override 문법). 카페24 공식·실측으로 확인된 지시어는 **괄호 단일태그**뿐이다. (근거: `SKILL.md §1`·`evidence-labels.md`)

**올바른 지시어 (전부 단일태그):**
```html
<!--@layout(/layout/basic/layout.html)-->   <!-- 페이지 첫 줄: 공통 레이아웃 지정 -->
<!--@contents-->                            <!-- 레이아웃 안: 본문 삽입 위치 (괄호 없음) -->
<!--@css(/_nk/css/nk.css)-->                <!-- 외부 CSS 로드 -->
<!--@js(/_nk/js/nk.js)-->                   <!-- 외부 JS 로드 -->
<!--@import(/_nk/inc/header.html)-->        <!-- 외부 HTML 조각 인클루드 -->
```

- **모듈은 지시어가 아니라 `module="모듈명"` HTML 속성**으로 구동한다. (예: `<div module="product_listnormal">...</div>`)
- **인라인 CSS가 필요하면** 일반 `<style>` 태그를 쓰거나(권장은 `<!--@css(경로)-->`로 외부 파일 분리). `<!--@css-->` 쌍태그는 없다.

## 6. 모디파이어 체이닝 실전 예시

여러 모디파이어를 `|`로 연결해 한 번에 처리:

```html
<!-- 상품명: 30자로 자르고, HTML 태그 제거 후 이스케이프 처리 -->
{$상품명|strip_tags|escape|cut:30}

<!-- 가격: 숫자 포맷 + 줄바꿈 없이 표시 -->
{$상품가격|number_format}원

<!-- 날짜: 포맷 변환 후 20자 제한 -->
{$등록일시|date_format:'Y년 m월 d일'|cut:20}

<!-- 상품 요약: 태그 제거 → 줄바꿈 처리 → 100자 제한 -->
{$상품요약설명|strip_tags|nl2br|cut:100}
```

**체이닝 적용 순서:** 왼쪽에서 오른쪽으로 순차 처리. `|strip_tags|cut:50`이면 태그 제거 후 50자 자름.

## 7. 자주 발생하는 실수 & 대처법

| 실수 | 원인 | 해결 |
|------|------|------|
| `|cut`으로 잘렸는데 `...`이 안 붙음 | 원본 값이 이미 N자 이하일 때는 잘리지 않음 (정상 동작) | 의도한 동작 맞음 |
| `|nl2br` 써도 줄바꿈 안 됨 | 원본 데이터에 실제 `\n`이 없고 `<br>` 태그가 이미 있는 경우 | 적용 불필요 |
| `|date_format` 오동작 | 포맷 문자열을 큰따옴표(`"`)로 감쌌을 때 | 반드시 작은따옴표(`'`) 사용 |
| `|resize` 적용 후 이미지 깨짐 | 외부 CDN URL에 적용 시 카페24 서버가 처리 못함 | 카페24 내부 이미지 URL에만 사용 |
| `|display` 써도 숨김 안 됨 | 속성 위치가 아닌 컨텐츠 위치에 작성함 | `<div {$변수|display}>` 형태로 태그 안에 작성 |
| 조건 분기를 `<!--[if]-->`로 시도 | `<!--[if]-->`는 미작동(양쪽 다 출력) | `\|display` 필터 또는 JavaScript로 |
