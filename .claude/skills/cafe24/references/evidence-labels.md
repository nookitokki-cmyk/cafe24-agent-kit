# 근거 라벨 정합성 — 공식 vs 실측 (단일 기준)

> kit 문서가 카페24 동작을 단정투로 쓰면 신뢰·유지보수가 깨진다. 이 표가 **어디까지가 [공식] 문서 근거이고, 어디부터가 [실측/관측]인지**의 단일 기준이다.
> 근거: `agent-kit/brain/_evidence/_evidence/`의 카페24 developers 문서 스크랩(2026-06 수집) 정독.
> 라벨: `[공식]` developers.cafe24.com 문서에 문장으로 있음 · `[실측]` kit 실무·라이브 관측(공식 문장 없음) · `[검증필요]` 근거 불충분.

## A. 공식 확정 (developers 문서 인용됨)
| 사실 | 근거 |
|---|---|
| 화면 4영역(헤더·푸터·바디·내비) | sdsupport/basic STEP01 |
| 레이아웃 = 로고·카테고리·푸터 공통, `<!--@layout()-->` + `<!--@contents-->`, 페이지별 레이아웃 교체 | basic STEP01 |
| 레이아웃 HTML 필수 = `<!DOCTYPE><html><head><body>` | basic |
| 모듈 = HTML+변수 조합, `module="id"`로 구동, **모듈별 변수는 정해져 있음** | basic STEP03 |
| 반복 우선순위 `$only_html > $count > DB`, **반복단위가 다르면 "마지막 블록이 count만큼 반복"** | basic |
| `xans-record-` + `anchorBoxId_N` = 렌더된 상품목록 반복단위(실측 렌더 마크업) | product_list cate_no_61 |

## B. 실측·관측 (공식 문장 없음 — 단정 금지, [실측] 라벨 필수)
| 관측 | 상태 |
|---|---|
| `<!--@css(...)-->`·`@js`·`@import` 지시어 문자열 | **[실측]** — 공식 basic/editor 본문에서 0건(오직 `@layout`만). 동작은 kit 관측. |
| `ec-base` / `ec-base-product` 클래스 | **[실측]** — 공식 근거 0건 |
| 스킨 CSS `optimizer.php` 번들링·캐시 지연(2~5분) | **[실측]** — 공식 문장 없음(문서사이트 자체 로더로만 등장) |
| `@css` 파일 없으면 base 폴더 폴백 로드(F22) | **[실측]** |
| EZ: `data-ez-*`·`sde_design/base` 경로·`sub_theme/add_theme` late-load·Easy는 SFTP 대량수정 불가(F35) | **[실측]** — Easy 전용 공식 가이드 자체가 수집분에 부재 |
| `xans-record-`가 스킨 소스엔 없고 렌더타임 주입 | **[실측]** — P0에서 소스 234p 중 0건 확인 |

## B-2. 2026-07-06 skin2 라이브 검증 (실측 확정 — `[검증됨]` = 가장 강한 근거)
> 라이브에서 직접 렌더·클릭해 확인. 문서 스크랩보다 우선한다.

| 관측 | 결과 |
|---|---|
| 표시 모듈(`product_list*`) 완전 재작성 | 카드 그리드·백지 재구축·전면 브랜드 리디자인 성공 **[검증됨]** |
| 구매버튼(`product_action`) 재마크업 | `onclick="{$action_*}"`·`link-*` 1:1 이식 시 장바구니 담기·주문 라우팅 성공 **[검증됨]** |
| 거래 표(`Order_*` 상품줄) de-table **금지** | 스킨에 `<form>` 없음 → 엔진이 `module=` 위치 보고 런타임에 폼·hidden·`name` 주입 + `rowspan`·배송그룹=table 경계. **표 뜯으면 주문 깨짐** **[검증됨]** |
| 카트 완전 커스텀 = JS 재구축 | 카트 숨기고 JS로 데이터 읽어 커스텀 UI + 숨긴 원본에 배선 연결 → 렌더+주문 성공 **[검증됨·프로토타입]** |
| `<!--[if]-->` 조건문 | **미작동**(양쪽 분기 다 출력). 조건분기는 `\|display` 필터 **[검증됨]** |
| 스코프밖/미정의 변수 | **빈 값**(문자 그대로 아님) **[검증됨]** |
| `{$product_name}` | 카페24가 `<span style>`로 감쌈 → `\|cut` 쓰면 마크업 잘려 깨짐 **[검증됨]** |
| `$count` 반복 제어 · `\|number_format` · `\|display` · `{$product_price}` | 작동 **[검증됨]** |
| `{$key}`/`{$no}` 인덱스 · `{$mall.*}` 점 형태 | product_listnormal에서 **빈 값** **[검증됨: 미작동]** |
| `|resize:W:H` | 몰에 실제 상품 사진 없어 확인 불가 **[검증필요]** |

## C. 검증 필요 (근거 불충분 — 추측 금지)
- **모듈 로직 보존한 채 마크업 재작성이 안전한 범위** → 표시 모듈·구매버튼은 **B-2에서 [검증됨]**, 거래 표는 골격 보존. 나머지(게시판·회원폼·옵션상품 등)는 모듈별 실증 필요. (`module-remarkup.md` §5)
- **"되는 모듈 vs 제약 모듈" 구분** → 공식 표 없음. `module-safety.json`은 [실측]. 핵심 경계선(표시=완전재작성 / 거래표=골격보존)은 **B-2에서 [검증됨]**.
- **베이직 스킨의 개별 페이지 파일 전수 목록** → 공식 미제공. `page-module-map.md`는 1개 몰 실측.

## 적용 규칙
1. 새 kit 문서는 B/C 항목을 단정투("~이다")로 쓰지 말고 **`[실측]`/`[검증필요]`를 붙인다.**
2. 기존 문서(SKILL.md·skin-method-detect.md 등)에서 `@css`·`ec-base`·optimizer를 공식처럼 서술한 대목은 **라벨 교정 대상**(리뷰 포인트 — 이번엔 미수정, 한비님 검토 후 일괄 반영 권장).
