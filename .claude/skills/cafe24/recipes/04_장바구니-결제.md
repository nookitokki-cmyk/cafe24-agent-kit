# 장바구니·결제 — 담기 → 주문서 작성 플로우

## 이게 뭔가요?
고객이 상품을 장바구니에 담은 뒤, 주문서를 작성하고 결제까지 이어지는 두 단계 페이지입니다.
장바구니 페이지에서는 수량 변경, 옵션 확인, 총 금액 확인이 가능하고,
주문서 페이지에서는 받는 사람 정보, 결제 수단, 포인트 적용 등을 입력합니다.
이 두 페이지를 합쳐서 "구매 플로우"라고 부릅니다.

## 어떤 페이지에서 쓰나요?
- 장바구니 페이지: `order_basket.html`
- 주문서 작성 페이지: `order_detail.html`
두 파일에 각각 코드를 넣어야 합니다.

## 필요한 모듈 (순서대로)

### 장바구니 페이지 (`order_basket.html`)

| 모듈 ID | 역할 | 필수여부 |
|---------|------|---------|
| `Order_Empty` | 장바구니가 비어 있을 때 표시할 내용 | 필수 |
| `Order_NormNormal` | 일반 상품 목록 (이미지·이름·옵션·수량·금액) | 필수 |
| `Order_list` | 장바구니 전체 금액 합산 표시 | 필수 |

### 주문서 페이지 (`order_detail.html`)

| 모듈 ID | 역할 | 필수여부 |
|---------|------|---------|
| `Order_form` | 주문자 정보 + 배송지 + 결제 수단 입력 폼 | 필수 |

## 복사해서 쓰는 HTML

### [A] 장바구니 페이지 (`order_basket.html`)

```html
<!--@@--
페이지 타입: 장바구니 (order_basket.html)
레시피: 04_장바구니-결제 (A)
--@@-->

<div class="nk-basket-wrap">
  <h1 class="nk-page-title">장바구니</h1>

  <!-- ① 장바구니가 비어있을 때 -->
  <module name="Order_Empty">
    <div class="nk-basket-empty">
      <p class="nk-empty-msg">장바구니에 담긴 상품이 없습니다.</p>
      <a href="/" class="nk-btn-continue">쇼핑 계속하기</a>
    </div>
  </module>

  <!-- ② 장바구니 상품 목록 -->
  <module name="Order_NormNormal">
    <!-- 상품 목록 헤더 -->
    <div class="nk-basket-header">
      <span class="nk-col-check">
        <input type="checkbox" id="allCheck" onclick="basketAllCheck(this.checked)">
      </span>
      <span class="nk-col-product">상품 정보</span>
      <span class="nk-col-qty">수량</span>
      <span class="nk-col-price">금액</span>
      <span class="nk-col-del">삭제</span>
    </div>

    <!-- 개별 상품 행 -->
    <ul class="nk-basket-list">
      <li module="item" class="nk-basket-item">
        <!-- 체크박스 -->
        <span class="nk-col-check">
          <input type="checkbox" name="basket_product_no[]"
            value="{$basket_product_no}" checked>
        </span>
        <!-- 상품 이미지 + 이름 + 옵션 -->
        <div class="nk-col-product">
          <a href="{$product_url}">
            <img src="{$thumbnail_url}" alt="{$product_name}" class="nk-basket-thumb">
          </a>
          <div class="nk-basket-product-info">
            <p class="nk-basket-product-name">{$product_name}</p>
            <!-- 선택한 옵션 이름 표시 -->
            <p class="nk-basket-option-name">{$option_name}</p>
          </div>
        </div>
        <!-- 수량 변경 -->
        <div class="nk-col-qty">
          <div class="nk-qty-control">
            <button type="button"
              onclick="changeBasketQty('{$basket_product_no}', -1)">−</button>
            <input type="text" value="{$quantity}"
              id="quantity_{$basket_product_no}" class="nk-qty-input">
            <button type="button"
              onclick="changeBasketQty('{$basket_product_no}', 1)">+</button>
          </div>
        </div>
        <!-- 상품 금액 -->
        <div class="nk-col-price">
          <span class="nk-basket-price">{$product_price|number_format}원</span>
        </div>
        <!-- 삭제 버튼 -->
        <div class="nk-col-del">
          <button type="button" class="nk-btn-del"
            onclick="deleteBasketItem('{$basket_product_no}')">×</button>
        </div>
      </li>
    </ul>
  </module>

  <!-- ③ 합계 금액 + 주문 버튼 -->
  <module name="Order_list">
    <div class="nk-basket-summary">
      <div class="nk-summary-row">
        <span class="nk-summary-label">상품 합계</span>
        <span class="nk-summary-value">{$product_price_total|number_format}원</span>
      </div>
      <div class="nk-summary-row">
        <span class="nk-summary-label">배송비</span>
        <span class="nk-summary-value">{$shipping_fee|number_format}원</span>
      </div>
      <div class="nk-summary-row nk-summary-total">
        <span class="nk-summary-label">결제 예정 금액</span>
        <strong class="nk-summary-total-price">{$total_price|number_format}원</strong>
      </div>
      <!-- 주문서 작성 버튼 -->
      <button type="button" class="nk-btn-order" onclick="doOrder()">
        주문하기 ({$total_price|number_format}원)
      </button>
      <!-- 쇼핑 계속하기 -->
      <a href="/" class="nk-btn-continue-link">쇼핑 계속하기</a>
    </div>
  </module>

</div>
```

### [B] 주문서 작성 페이지 (`order_detail.html`)

```html
<!--@@--
페이지 타입: 주문서 작성 (order_detail.html)
레시피: 04_장바구니-결제 (B)
--@@-->

<div class="nk-order-wrap">
  <h1 class="nk-page-title">주문서 작성</h1>

  <!-- 주문 폼 전체 (카페24가 자동으로 채워주는 모듈) -->

    <form id="orderForm" method="post">

      <!-- 주문 상품 목록 (읽기 전용 확인) -->
      <div class="nk-order-section">
        <h2 class="nk-order-section-title">주문 상품</h2>
        <ul class="nk-order-product-list">
          <li module="item" class="nk-order-product-item">
            <img src="{$thumbnail_url}" alt="{$product_name}" class="nk-order-thumb">
            <div class="nk-order-product-detail">
              <p class="nk-order-product-name">{$product_name}</p>
              <p class="nk-order-option">{$option_name}</p>
              <p class="nk-order-qty">{$quantity}개</p>
            </div>
            <span class="nk-order-item-price">{$product_price|number_format}원</span>
          </li>
        </ul>
      </div>

      <!-- 주문자 정보 -->
      <div class="nk-order-section">
        <h2 class="nk-order-section-title">주문자 정보</h2>
        <div class="nk-form-group">
          <label class="nk-form-label">이름</label>
          <input type="text" name="order_name" value="{$order_name}"
            class="nk-form-input" placeholder="이름을 입력하세요">
        </div>
        <div class="nk-form-group">
          <label class="nk-form-label">연락처</label>
          <input type="tel" name="order_phone" value="{$order_phone}"
            class="nk-form-input" placeholder="010-0000-0000">
        </div>
        <div class="nk-form-group">
          <label class="nk-form-label">이메일</label>
          <input type="email" name="order_email" value="{$order_email}"
            class="nk-form-input" placeholder="email@example.com">
        </div>
      </div>

      <!-- 배송지 -->
      <div class="nk-order-section">
        <h2 class="nk-order-section-title">배송지</h2>
        <div class="nk-form-group">
          <label class="nk-form-label">받는 분</label>
          <input type="text" name="shipping_name" value="{$shipping_name}"
            class="nk-form-input">
        </div>
        <div class="nk-form-group">
          <label class="nk-form-label">주소</label>
          <div class="nk-addr-row">
            <input type="text" name="shipping_zipcode" value="{$shipping_zipcode}"
              class="nk-form-input nk-zipcode" readonly>
            <button type="button" onclick="searchZipcode()" class="nk-btn-zipcode">
              우편번호 찾기
            </button>
          </div>
          <input type="text" name="shipping_addr" value="{$shipping_addr}"
            class="nk-form-input" readonly>
          <input type="text" name="shipping_addr_detail" value="{$shipping_addr_detail}"
            class="nk-form-input" placeholder="상세 주소 입력">
        </div>
        <!-- 배송 메모 -->
        <div class="nk-form-group">
          <label class="nk-form-label">배송 메모</label>
          <select name="shipping_memo" class="nk-form-select">
            <option value="">배송 요청사항을 선택하세요</option>
            <option value="문 앞에 놓아주세요">문 앞에 놓아주세요</option>
            <option value="경비실에 맡겨주세요">경비실에 맡겨주세요</option>
            <option value="직접 입력">직접 입력</option>
          </select>
        </div>
      </div>

      <!-- 결제 금액 요약 -->
      <div class="nk-order-section nk-payment-summary">
        <h2 class="nk-order-section-title">결제 금액</h2>
        <div class="nk-summary-row">
          <span>상품 합계</span>
          <span>{$product_price_total|number_format}원</span>
        </div>
        <div class="nk-summary-row">
          <span>배송비</span>
          <span>{$shipping_fee|number_format}원</span>
        </div>
        <div class="nk-summary-row nk-summary-total">
          <strong>최종 결제 금액</strong>
          <strong class="nk-total-highlight">{$total_price|number_format}원</strong>
        </div>
      </div>

      <!-- 결제 버튼 -->
      <div class="nk-order-actions">
        <button type="submit" class="nk-btn-pay">결제하기</button>
      </div>

    </form>
  </module>
</div>
```

## 여기만 바꾸세요

| 바꿀 부분 | 현재 값 | 설명 |
|-----------|---------|------|
| 배송 메모 항목 | `문 앞에 놓아주세요` 등 | 자주 쓰는 배송 메모로 교체 |
| `결제하기` 버튼 텍스트 | `결제하기` | 원하는 문구로 변경 |
| `nk-btn-pay` 배경색 | `#111` | 브랜드 컬러로 변경 |

> **`returnUrl`이 뭔가요?**
> 결제 완료 후 어느 페이지로 돌아올지 지정하는 URL입니다. 카페24가 자동 처리하므로 HTML에서 직접 건드릴 필요는 없습니다. 관리자 → [주문] → [결제 설정]에서 완료 페이지를 지정하세요.

> **비회원 주문은?**
> 카페24는 비회원 주문을 관리자 → [회원] → [회원 정책]에서 허용/차단할 수 있습니다. 비회원 허용 시 `Order_form` 모듈이 자동으로 로그인 없이도 폼을 보여줍니다.

## 자주 하는 실수

1. **수량 변경 버튼이 눌려도 숫자가 안 바뀌어요**
   → `changeBasketQty()` 함수는 카페24 기본 JS에서 제공합니다. 스킨 `<head>` 안에 카페24 공통 JavaScript가 로드되어 있는지 확인하세요. (`<script src="/js/common_tag.js">` 등)

2. **주문서에서 주소 검색 팝업이 안 열려요**
   → `searchZipcode()` 함수도 카페24 기본 JS에 있습니다. 공통 JS 누락이 원인인 경우가 대부분입니다.
