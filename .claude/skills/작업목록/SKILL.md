사용자가 `/작업목록` 을 요청했습니다.

## 원칙

- 상태의 진실원본은 각 클라이언트 폴더의 `.workflow.md` 입니다.
- `.workspace-index.json` 은 `/작업목록` 실행 때 다시 만드는 파생 목록입니다.
- 이 명령은 read-only 조회 명령입니다. 카페24 서버, SFTP, API, `src/` 파일을 수정하지 않습니다.

## 실행

키트 루트에서:

```bash
cd mcp
python cli.py workspace-list
```

## 보고 형식

결과 JSON의 `workspaces[]` 를 쉬운 표로 요약합니다.

| 항목 | 표시 |
|---|---|
| 몰 ID | `mall_id` |
| 클라이언트 | `client_name` |
| 현재 상태 | `current_status` |
| 다음 액션 | `next_action` |
| 마지막 업데이트 | `last_updated` |

## 사용자가 이어가고 싶어하면

1. 작업이 1개면 해당 몰 ID를 다시 확인합니다.
2. 작업이 2개 이상이면 번호 또는 몰 ID를 선택하게 합니다.
3. 선택 후에도 바로 업로드하지 않습니다.
4. 실제 수정은 기존 정규 명령(`/접속세팅`, `/카페24-워크플로우`, `/검증`)으로 이어갑니다.

## 완료 예시

```text
[작업목록]
- 기준: clients/*/.workflow.md
- 작업 수: {N}
- 다음: 이어갈 몰 ID를 선택하세요. 업로드는 /업로드준비 전까지 하지 않습니다.
```
