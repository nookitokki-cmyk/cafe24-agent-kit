사용자가 `/요소측정` 을 요청했습니다. **코드를 고치지 않는다.** 숫자와 사실만 모은다.

## 전제

`/접속세팅` 이 끝났거나, 최소한 **skin_code·대상 URL** 이 있어야 한다. 없으면 `/접속세팅` 먼저.

## 질문 대본 (하나씩)

1. 「**어느 페이지**를 바꿀까요? (메인 / 상품상세 / URL 붙여넣기)」
2. 「**레퍼런스**가 있나요? (Figma 링크 / 스크린샷 / 다른 사이트 URL)」
3. 「**제목(헤드라인) 폰트**는 무엇인가요? (모르면 '모름' — 그럼 레퍼런스에서 측정할게요)」
4. 「**본문 폰트**는요?」
5. 「**제목과 본문 사이 여백**을 px로 알려주세요. (모르면 스크린샷에서 잰 값을 제안할게요)」
6. 「**섹션 위아래 여백** (px)?」
7. 「**메인 색** (hex 또는 '레퍼런스와 동일')?」
8. 「**PC만** 인가요, **모바일(360px)** 도 같이 인가요?」

## 에이전트가 할 수 있는 측정

- 브라우저·스크린샷 있으면: padding, font-size, color **숫자로 제안**
- MCP 있으면: `cafe24_read_page` / `cafe24_sftp_read` 로 **현재 값** 읽고 표로 제시

## 출력 형식

```markdown
## 실측 시트
- 페이지: ...
- 폰트: 제목 / 본문
- 여백: ...
- 색상: ...
- PC/모바일: ...
- 레퍼런스: ...
```

마지막: 「이 숫자로 진행할까요? (예/수정)」→ **예** 받으면 `/디자인수정` 또는 워크플로 진행.
