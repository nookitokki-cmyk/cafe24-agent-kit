# 시안 제출 체크리스트 (클라이언트·작업 의뢰 시)



> 에이전트 `/레퍼런스인입` 이 **물을 수 있는 형태**로만 제출하면 됩니다.  

> **레퍼런스 URL** 또는 **시안 파일** — **둘 중 하나만** 있어도 됩니다.



---



## 공통 (항상)



```

□ 타겟 몰 ID (예: ecudemo400786) — 어디에 올릴지

□ 파트너 FTP / SFTP 접속 가능 (협업자·에이전트와 /접속세팅)

```



## 협업자·에이전트 (선택 — PC+MO 동일 템플릿 시 권장)



```

□ 「모바일 전용 디자인 사용설정」 사용안함 확인됨 (CAFE24.MOBILE_WEB=false)

  — 관리자: 쇼핑몰 설정 → 사이트 설정 → 쇼핑몰 환경 설정 → 모바일 → 기본설정 → 사용설정

  — MCP로 변경 불가; 작업 시작 전·배포 후 작업자 확인 요청 (`rules/responsive-mobile.md`)

```



---



## A. 레퍼런스 URL만



```

□ 레퍼런스 쇼핑몰 URL (예: https://ecudemo393674.cafe24.com/)

□ (선택) 특정 페이지만 줄이기 — 없으면 「전 페이지」

□ (선택) 데이터: 디자인만 / 배너·상품까지

```



**하지 말 것:** 「예쁘게 맞춰줘」만 보내기



---



## B. 정적 HTML / CSS / JS



```

□ zip 또는 폴더: index.html, 서브 HTML, css/, js/, images/

□ PC (+ 가능하면 모바일) 화면

□ HANDOFF.md (아래 10줄)

```



**HANDOFF.md 예시:**



```markdown

- 범위: 메인, PLP, PDP, 로그인, 장바구니

- 폰트: Bricolage Grotesque / 13px

- breakpoint: 1440 / 390

- [정적] About, Contact

- [카페24 module] PLP, PDP, basket, login

```



---



## C. Figma



```

□ Figma 링크 (보기 권한)

□ 프레임명 = 페이지명 (Main, PLP-Men, Login …)

□ breakpoint 메모 (1440 / 768 / 390)

□ (있으면) Variables / Components 페이지

□ 인터랙션: 메뉴, 스크롤 헤더, 팝업

```



---



## A + B 둘 다 있을 때



- **시안(B)** = 디자인·레이아웃 기준

- **URL(A)** = 카페24 module·동작·데이터 참고



---



## 진행 규칙 (작업자용)



1. 에이전트·협업자가 **페이지 목록·실측표**를 만들면 검토

2. **「예」하기 전에는** 코드·FTP 수정 없음

3. 범위 기본값 = **레퍼런스/시안에서 확인 가능한 페이지 전부**

4. **수치를 모를 때:** 에이전트가 레퍼런스 실측 기본값으로 시안을 먼저 보여줄 수 있음 (작업자 동의 후). 기본값은 `getComputedStyle` + 스크린샷 **dual-source** (`/레퍼런스인입` Q5 이스케이프)
