# 실측 격차가 생기는 이유와 대응법



> **대상:** Cafe24 EZ 스킨 리뉴얼 작업자  

> **전제:** 레퍼런스 몰 vs 내 몰을 Playwright·DevTools로 비교할 때 숫자가 시안과 다르게 나오는 경우



---



## 1. AI·실측 수치가 시안(mockup)과 다른 이유



### 1-1. CSS 레이어 전쟁



카페24 EZ skin16은 **기본 CSS → optimizer 번들 → 커스텀 `_ref` 레이어** 순으로 쌓입니다.



| 레이어 | 예시 | 작업자가 흔히 놓치는 점 |

|--------|------|------------------------|

| EZ 기본 | `layout.css` `#container #contents { width:92% }` | margin만 0으로 바꿔도 width는 92% 유지 |

| optimizer | `optimizer_user.css` PLP/PDP 번들 | 시안에는 없는 `line-height`, `font-size` 잔재 |

| 커스텀 | `_ref393674/css/sub-product.css` | 나중에 쓴 규칙이 이겨야 하는데 specificity 부족 |



**비유:** 시안은 「의도」이고, 라이브는 「레이어 전쟁」입니다. 한 장의 PNG와 실제 DOM+CSS 스택은 다릅니다.



### 1-2. `#contents` 92% trap (모바일 390px)



- viewport 390px일 때 `#contents` computed width ≈ **359px** (92%)

- 히어로·배너 자식에 `width:100vw`만 걸어도 **부모가 359px**이면 좌우 흰 여백이 남습니다.

- **해결:** `#container #contents { width:100% !important }` (ID 2개, 페이지 타입별)



### 1-3. 폰트 스택 fallback



| 항목 | 레퍼런스 | 타겟 EZ 기본 |

|------|----------|--------------|

| 본문 | Bricolage Grotesque 13px | Jost 12~13px |

| line-height | 13×1.4 ≈ **18.2px** | 12×1.5 = **18px** 또는 더 넓음 |



같은 `line-height:1.4`라도 **font-size가 12px vs 13px**이면 줄 간격·카드 높이가 달라집니다.



### 1-4. 페이지 타입 혼동



| 타입 | container | `#contents` | 예 |

|------|-----------|-------------|-----|

| PLP 풀폭 | max-width 100%, padding 50/20/100 | `display:contents` 권장 | product/list |

| narrow | max-width 1200px 중앙 | width 100% inside | login, board |



PLP용 CSS를 narrow 페이지에 쓰면 「의도와 다른 여백」이 생깁니다. **인입 단계 페이지 타입 표**를 먼저 채우세요.

## 2. 격차를 발견했을 때 체크리스트



1. **DevTools → Computed** 양쪽(레퍼런스·내 몰)에서 같은 셀렉터 비교  

   - `.description`, `.name`, `.spec`, `li.item` — `line-height`, `margin`, `padding`, `font-size`

2. **overflow만 보지 말 것** — overflow 없어도 92% width·line-height 차이로 「안 맞아 보임」

3. **페이지 타입 표** 확인 — PLP full vs narrow 구분 후 해당 `sub-*.css`만 수정

4. **100점 verify 루프** — `python work/scripts/ref393674-score-{phase}.py`  

   - **100 미만 = 미완료** (90점도 FAIL)

5. **데이터 vs HTML** 구분  

   - 카테고리 배너 이미지·map 텍스트 → 관리자 업로드 **또는** 템플릿 placeholder  

   - menupackage 항목 수 → 몰 카테고리 트리 데이터

## 3. 설명할 때 쓸 말 (선택)



- 「시안 픽셀과 라이브 픽셀은 다릅니다. EZ 기본 CSS가 먼저 깔려 있어요.」

- 「margin 0만 넣고 끝내면 `#contents`는 여전히 92%입니다. width를 이겨야 해요.」

- 「상품 카드 `.description` line-height는 매 프로젝트에서 터집니다. ref와 tgt 둘 다 getComputedStyle 하세요.」

- 「PLP는 `#contents` 안에 모듈이 들어가도, `display:contents`로 ref처럼 container 직계처럼 보이게 맞출 수 있어요.」

- 「배너 `map` 텍스트는 이미지맵 HTML입니다. 관리자에 배너 없으면 placeholder가 필요해요.」

- 「score 스크립트가 100점이 아니면 제출 전입니다. 90점은 이제 없습니다.」

- 「모바일은 `/m/` URL 말고 메인 URL에 viewport 390으로 검증하세요. 관리자에서 모바일 전용 디자인 **사용안함** 확인.」
