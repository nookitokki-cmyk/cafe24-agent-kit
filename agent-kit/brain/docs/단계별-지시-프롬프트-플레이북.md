## 0단계 — 온보딩
> "새 클라이언트야. 몰 ID는 ○○○." → `/새클라이언트` → `/접속세팅` (호스트·포트·아이디는 화면 값 그대로, 비밀번호는 채팅 금지)

## 1단계 — 레퍼런스 인입
> "https://○○○.cafe24.com/ /레퍼런스인입 — 레퍼런스를 분석하고 데이터를 저장해줘. 작업 시작 전에 /plan 세우고, 단계별로 이어서 진행할 수 있게 핸드오버까지 저장해줘."

## 2단계 — 합의 계획 + 자율 실행 (본작업 발사)
> "①레거시 코드 전부 정리하고 새 디자인에 영향 없게 자가 검증해 ②배너 이미지는 higgsfield ○○모델로 생성해(카탈로그에 없다고 단정 말고 검색해) ③섹션별로 비주얼 90점 넘어야 다음 진행 ④끝나면 /ultraqa로 전체 점검 ⑤이걸 /ralplan으로 합의 계획 세우고 /ultrawork로 진행해."

## 3단계 — 잔존 CSS 정밀 분석 (문서 의존 금지)
> "키트 문서에만 의존하지 말고 실제 코드베이스(css/module)를 전수 분석해서, 잔존 CSS가 새 스킨을 명시도 싸움에서 이기지 못하게 해줘. 페이지마다 [module CSS 선분석 → 충돌 매핑 → 처방 → 라이브 확인] 절차로. 이해했으면 요약해서 되물어봐."

## 4단계 — 모듈 HTML 재작성 (편집이 아니라 신규)
> "**기능은 그대로, 모양은 새로** — 모듈 기능(module 속성·변수·폼)은 보존하고 마크업은 레퍼런스 구조로 새로 짜줘. 깨지면 안 되는 동작: (장바구니·옵션선택·정렬·검색 등 나열). 끝나면 그 동작들 실제로 눌러서 확인하고 점수 보고해. 못 맞추는 모듈 나오면 임의 우회 말고 물어봐."

## 5단계 — 조직 편성 (규모 커질 때)
> "페이지 타입별로 하나씩 끝까지 진행해줘." (직렬 안전형)
> "멀티 오케스트레이션으로 상위 에이전트에 하위 에이전트 배치하는 그룹형으로, 어떻게 배치할지 /plan 세우고 진행하자." (병렬 고속형 — ⚠️ 세션 한도 리스크: 하위 스폰 제한·공유 브리프·재개(resume) 우선 가드레일 필수)

## 6단계 — 서버 전 페이지 확장 (레퍼런스에 없는 페이지 포함)
> "서버에 존재하는 페이지 템플릿을 **전수 인벤토리로 뜨고**(실페이지/조각/팝업/미사용 분류), 레퍼런스에 있으면 그 구조로, 없으면 design.md + 기존 타입 중 유사 패턴으로 전부 편집해줘. 페이지별 **처리 상태표(완료/수용/제외+사유)** 로 보고하고 **상태표에 빈 칸이 없어야 완료**야."

## 7단계 — 디테일·상태 검증 (지시 안 한 것까지)
> "모든 인터랙티브 요소는 **상태 매트릭스**(hover/focus/active/disabled/에러)까지 검증해. 그리고 검증 시작 전에 '무엇을 검증해야 완전한가' **체크리스트를 네가 먼저 만들어 보여주고**, 검증 못 한 항목은 '통과'가 아니라 **'미검증'으로 분류**해서 남겨."

## 8단계 — 숨은 구스타일 시각 스윕 (눈에 보이는 잔존)
> "전 페이지를 PC+모바일 스크린샷으로 뜨되 **드롭다운·모달·드로어 같은 '열림 상태'까지 포함**해서, design.md와 어긋나는 지점을 눈으로 전수 검사해 **결함 목록**(페이지·위치·증상·원인 추정)을 만들어. 내가 본 결함 ○○도 목록에 넣고, **목록이 수정→재검증으로 전부 소진될 때까지 반복**해."
