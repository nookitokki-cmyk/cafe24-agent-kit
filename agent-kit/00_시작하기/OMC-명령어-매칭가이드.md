## 쓰면 안 되는 때 (금지·주의)

| 하지 말 것 | 이유 | 대신 |
|------------|------|------|
| 인입 전 `/디자인수정` | 타입표·실측 없이 추측 px · PLP=narrow 오판 | `/레퍼런스인입` → 「예」 |
| 인입 중 FTP/HTML | `05-reference-intake` 게이트 위반 | 인입표 승인 후 |
| 실측 없이 「예쁘게」 | `디자인수정` 게이트 위반 | `/요소측정` |
| Easy 스킨에 HTML 자동 작업 | `editor_type` ≠ H | Easy 안내·전환 요청 |
| MO 검증을 `/m/` URL만 | 별도 mobile 스킨 오판 | 메인 URL + 390 viewport |
| `CAFE24.MOBILE_WEB=true` 상태 MO score | verify-loop 무의미 | 관리자 OFF 먼저 |
| 단계 건너뛰기 (`/카페24-워크플로우`) | 에이전트가 1·2차 거부 | 게이트 순서 준수 |

**예외:** 단일 페이지 quick-fix(색·문구 1곳)는 `/레퍼런스인입` 생략 가능 — 대신 `/요소측정`으로 숫자 명시.

## OMC vs agent-kit 명령 — 차이점

| 구분 | OMC (Oh My Claude) | agent-kit |
|------|-------------------|-----------|
| **위치** | Cursor 플러그인 `oh-my-claude` (wiki, notepad, trace 등) | `agent-kit/.claude/commands/*.md` |
| **슬래시 명령** | 플러그인 자체 명령 (프로젝트 무관) | **카페24 전용** 10개 `.md` 대본 |
| **에이전트** | OMC 서브에이전트 (architect, executor 등) | `카페24-워크플로우` · `카페24-도우미` (`.claude/agents/`) |
| **언제 OMC** | 멀티에이전트·메모리·위키·트레이스 | 카페24 스킨·FTP·verify-loop 작업 |
| **언제 agent-kit** | `/레퍼런스인입`, `/디자인수정` 등 | 항상 카페24 작업 시 |

**실무:** 카페24 작업은 **agent-kit 슬래시 명령**이 우선. OMC 플러그인은 세션 관리·병렬 에이전트 보조.

Cursor에서 agent-kit 폴더를 워크스페이스 루트로 열면 `.claude/commands/` 슬래시가 인식됩니다.

## 추천 콤보 플로우

### A. 레퍼런스 1:1

```
/접속세팅
  → /레퍼런스인입
  → (인입표 검토)
  → 「예」
  → /디자인수정  (또는 /카페24-워크플로우 → 03-reference-renewal)
  → Phase별 verify-loop (score ≥ 90, MO = 100)
  → FTP 배포 · 캐시 확인 · 관리자 재확인
```

```
/접속세팅 (미완 시)
  → /요소측정
  → 「예」
  → /디자인수정 (01-quick-fix)
```

### C. 처음·막힘

```
/도움말  (또는 /카페24-도와줘)
  → 00_시작하기 00~05
  → 02_막혔을때/함정-INDEX.md
```

```
/키트시작
```
