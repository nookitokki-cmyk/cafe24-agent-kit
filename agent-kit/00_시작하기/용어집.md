# 용어집 — 카페24 작업에서 만나는 말들

> **이 문서의 역할:** 이 키트로 카페24 쇼핑몰을 만들거나 고칠 때 화면이나 안내문에서 마주치는 낯선 단어를, 한 곳에서 쉽게 찾아보는 사전입니다.
> **읽는 법:** 전부 읽을 필요 없습니다. 막힌 단어 하나만 `Ctrl+F`로 찾아 그 줄만 보세요.
> **표기 규칙:** 모르는 값은 지어내지 않았습니다. "내 환경의 값은 관리자/발급 화면에서 확인"이라고 안내된 곳은 그대로 확인해 주세요.

이 사전의 각 항목은 [용어] / **쉬운 뜻** / **왜 중요한지·주의할 점** / (있으면) 더 볼 문서 순서입니다.

## 1. 내 쇼핑몰을 가리키는 값

### 몰 ID (mall ID)

- **쉬운 뜻:** 내 쇼핑몰 주소 앞부분 한 덩어리. 주소가 `https://ecudemo400786.cafe24.com/` 이면 `ecudemo400786` 이 몰 ID입니다.
- **왜 중요·주의:** 거의 모든 작업에서 "어느 쇼핑몰인지" 알려주는 이름표입니다. `cafe24.com` 전체를 통째로 적으면 안 되고 **앞부분만** 적습니다. 파트너 로그인 ID와 몰 ID가 다를 수 있으니 헷갈리지 마세요.
- **더 볼 것:** [02-몰ID-찾기.md](./02-몰ID-찾기.md)

### skin_no (스킨 번호) vs skin_code (스킨 코드) ⚠️ 다름 주의

- **쉬운 뜻:** 카페24는 디자인 한 벌을 "스킨(skin)"이라 부릅니다. 한 쇼핑몰에 여러 스킨이 있을 수 있어요.
  - `skin_no` = 그 스킨에 매겨진 **관리용 번호**.
  - `skin_code` = 실제 **파일이 들어 있는 폴더 이름** (예: `skin1`, `skin14`).
- **왜 중요·주의:** 둘이 **같은 숫자일 거라고 가정하면 안 됩니다.** `skin_no`가 14라고 폴더가 꼭 `skin14`인 건 아닙니다. 파일을 올릴 폴더 경로는 `skin_no`가 아니라 **`skin_code`** 기준입니다 (`/{skin_code}`). 정확한 값은 `cafe24_list_themes`(아래 MCP 참고)로 확인하세요.
- **더 볼 것:** [03-접속-웹FTP-vs-SFTP.md](./03-접속-웹FTP-vs-SFTP.md), [05-MCP-연결-개요.md](./05-MCP-연결-개요.md)

## 2. 파일을 올리고 내리는 길

### 웹 FTP vs 디자인 SFTP

- **쉬운 뜻:** 둘 다 쇼핑몰 서버에 디자인 파일을 올리고 내리는 "창고 문"입니다. 누구냐에 따라 문이 다릅니다.
  - **웹 FTP** = 파트너센터에서 만든 **샘플몰** 작업자가 주로 쓰는 문.
  - **디자인 SFTP** = 실제 **운영 중인 쇼핑몰** 관리자가 권한을 신청해서 쓰는 문. (SFTP = SSH라는 보안 통신으로 파일을 주고받는 방식)
- **왜 중요·주의:** 문이 다르면 **접속 정보(호스트·포트·프로토콜)도 다릅니다.** 예시 호스트나 "포트는 항상 22번" 같은 기본값을 그대로 쓰면 안 됩니다 — **카페24가 발급해 준 값 그대로**만 쓰세요. 파트너 샘플몰은 편집 경로가 운영 몰과 달라서, 운영 몰의 `/{skin_code}` 규칙을 그대로 적용하면 안 됩니다.
- **더 볼 것:** [03-접속-웹FTP-vs-SFTP.md](./03-접속-웹FTP-vs-SFTP.md)

### 파트너 샘플몰 vs 운영 몰

- **쉬운 뜻:**
  - **파트너 샘플몰** = 파트너센터에서 연습·제작용으로 만든 임시 쇼핑몰.
  - **운영 몰** = 실제 손님이 들어와 주문하는 진짜 쇼핑몰.
- **왜 중요·주의:** 작업 시작 전 **둘 중 무엇인지 먼저 정해야** 합니다. 접속 방법(웹 FTP / 디자인 SFTP), 편집 경로, 주의 수준이 전부 달라집니다. 운영 몰은 손님이 실시간으로 보고 있으므로 더 신중해야 하고, 백업이 필수입니다.
- **더 볼 것:** [00-아무것도-모를-때.md](./00-아무것도-모를-때.md), [03-접속-웹FTP-vs-SFTP.md](./03-접속-웹FTP-vs-SFTP.md)

## 3. 자동화로 연결하기 (선택)

### MCP

- **쉬운 뜻:** 에이전트(AI)에게 카페24를 다룰 수 있는 **리모컨**을 쥐여 주는 연결 방식입니다. 리모컨이 있으면 "스킨 목록 보여줘", "이 파일 읽어줘" 같은 일을 AI가 직접 할 수 있어요.
- **왜 중요·주의:** MCP 없이 카페24 파일관리자로 손수 해도 됩니다. MCP는 자동화를 편하게 해 주는 **선택 도구**입니다. 이 키트는 안전을 위해 **읽기 권한만**(아래 Scope) 기본으로 씁니다.
- **더 볼 것:** [05-MCP-연결-개요.md](./05-MCP-연결-개요.md), [../connect/MCP-OAUTH-GUIDE.md](../connect/MCP-OAUTH-GUIDE.md)

이 키트에서 자주 쓰는 MCP 도구:

| 도구 이름 | 하는 일 |
|-----------|---------|
| `cafe24_list_themes` | 스킨 목록 보기 (skin_no, skin_code 확인) |
| `cafe24_read_page` | 디자인 HTML 파일 한 개 읽기 |
| `cafe24_sftp_*` | SFTP로 파일 읽기·올리기 (연결 정보는 발급값으로) |

### OAuth · 토큰 (token)

- **쉬운 뜻:**
  - **OAuth** = 내 쇼핑몰이 "이 앱이 내 디자인을 읽어도 돼"라고 **허락하는 절차**.
  - **토큰** = 그 허락의 증표인 **출입증**. 출입증이 있어야 MCP가 동작합니다.
- **왜 중요·주의:** 앱을 개발자센터에 등록만 하고 **출입증(토큰) 발급 단계를 빼먹으면 MCP가 절대 안 됩니다.** 발급은 "터미널 → 브라우저 주소창에서 허용 → 터미널" 순서로 진행합니다. 출입증에는 수명이 있어요: 인증 code 1분, access_token 2시간, refresh_token 14일. `client_secret`이나 비밀번호는 **채팅·메신저에 절대 붙여넣지 마세요.**
- **더 볼 것:** [../connect/MCP-OAUTH-GUIDE.md](../connect/MCP-OAUTH-GUIDE.md), [05-MCP-연결-개요.md](./05-MCP-연결-개요.md)

### Scope (권한 범위) — mall.read_design

- **쉬운 뜻:** 앱이 내 쇼핑몰에서 **어디까지 손댈 수 있는지** 정하는 권한 설정. `mall.read_design`은 "디자인을 **읽기만** 가능"이라는 뜻입니다.
- **왜 중요·주의:** 이 키트는 최소 권한 원칙으로 **읽기만** 켭니다. 파일 수정은 API 쓰기 권한이 아니라 **FTP/SFTP**로 합니다. 상품·주문·회원 같은 불필요한 권한까지 켜면 보안·심사 위험이 커지니 끄세요.
- **더 볼 것:** [05-MCP-연결-개요.md](./05-MCP-연결-개요.md)

## 4. 카페24 디자인 시스템 — 두 갈래

### 스마트디자인 (HTML 방식)

- **쉬운 뜻:** HTML/CSS 코드를 직접 손으로 편집하는 카페24의 정통 디자인 방식. 이 키트가 다루는 **주력 방식**입니다.
- **왜 중요·주의:** 이 방식이어야 코드로 자유롭게 디자인할 수 있습니다. 작업 시작 전 내 스킨이 이 방식(아래 editor_type **H**)인지 확인하세요.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md), [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

### 스마트디자인Easy (EZ, data-ez 방식)

- **쉬운 뜻:** 코드 대신 **마우스로 블록을 끌어다 놓는 비주얼 편집기**(Easy). HTML 안에 `data-ez-...` 같은 특수 표시가 붙어 있고, 카페24 관리자 "디자인 편집" 화면과 연결됩니다.
- **왜 중요·주의:** `data-ez-*`, `ez-prop`, `ez-var` 같은 표시는 **EZ 전용**이라 함부로 지우면 관리자 편집 화면이 깨지거나 화면이 안 보일 수 있습니다. Easy 스킨을 HTML 방식으로 바꾸는 작업(EZ 제거)은 **사용자가 분명히 그러라고 할 때만** 합니다.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md)

### editor_type (H / E / D / W / C)

- **쉬운 뜻:** 내 스킨이 어떤 편집 방식인지 알려주는 **분류 글자**입니다.
  - **H** = 스마트디자인(HTML 방식) → 이 키트의 코드 작업 대상.
  - **E** = 스마트디자인Easy(비주얼 편집).
  - **D / W / C** = 그 밖의 방식.
- **왜 중요·주의:** **H가 아니면** 이 키트의 HTML 코드 작업 흐름을 그대로 적용하면 안 됩니다. 작업 전 `cafe24_list_themes`로 `editor_type`을 꼭 확인하세요. 단, **E라도 SFTP 폴더 접근이 될 수 있어서** "Easy면 무조건 SFTP 불가"라는 말은 과장입니다 — 실제로 확인하세요.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md), [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

## 5. 카페24 코드 문법

### module (모듈)

- **쉬운 뜻:** 카페24가 상품·게시판·로그인 같은 **실제 데이터를 끼워 넣어 주는 기능 블록**입니다. 코드에서 `<div module="product_listnormal">` 처럼 씁니다.
- **왜 중요·주의:** 모듈 안쪽 구조를 마음대로 바꾸면 데이터 연결(바인딩)이 깨집니다. 스타일(색·글꼴)만 바꾸거나, 모듈 **바깥**에 감싸는 칸을 추가하는 식으로 작업하세요. 아래 치환변수는 **모듈 안에서만** 작동합니다.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md)

### 치환변수 {$...}

- **쉬운 뜻:** 카페24가 화면을 그릴 때 **실제 값으로 바꿔 넣어 주는 빈칸**입니다. 예: `{$mall_name}` → 실제 쇼핑몰 이름, `{$product_name}` → 실제 상품명.
- **왜 중요·주의:** **반드시 `module="..."` 안에서만 진짜 값으로 바뀝니다.** 모듈 밖에 `{$mall_name}`을 쓰면 값이 채워지지 않아 **빈 값(화면에 아무것도 안 보임)**이 되거나 모듈 연결이 깨집니다. 모듈 밖에서 변하지 않는 값은 직접 글자로 적어(하드코딩) 두세요. (라이브 검증 2026-07-06: 스코프 밖 변수는 `{$mall_name}` 글자 그대로가 아니라 **빈 값**으로 나옴)
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md)

### 지시어 (`<!--@...-->`)

- **쉬운 뜻:** 카페24 엔진에게 "여기에 무엇을 끼워라"라고 시키는 **특수 명령 주석**입니다. 예: `<!--@import(...)-->`(다른 HTML 조각 불러오기), `<!--@css(...)-->`(CSS 불러오기), `<!--@layout(...)-->`(레이아웃 지정), `<!--@contents-->`(본문 들어갈 자리).
- **왜 중요·주의:** 겉보기엔 평범한 주석이라 **편집기(linter)가 경로 오타를 못 잡습니다.** 한 글자만 틀려도 헤더·푸터가 통째로 사라질 수 있으니 경로는 **눈으로 직접** 확인하세요. 특히 `@css`에는 `?v=1` 같은 물음표 쿼리를 붙이면 안 됩니다(파일을 못 찾아 통째로 빠짐).
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md), [02-몰ID-찾기.md](./02-몰ID-찾기.md)

## 6. 이 키트만의 작업 약속

### nk- 접두사

- **쉬운 뜻:** 이 키트에서 새로 만드는 모든 CSS 클래스 이름 앞에 붙이는 **꼬리표**입니다. 예: `nk-header`, `nk-product-card`.
- **왜 중요·주의:** 카페24 원래 코드와 새로 만든 코드를 **한눈에 구분**하기 위한 약속입니다. 새 클래스에는 반드시 `nk-`를 붙이세요. 그래야 나중에 무엇이 내 작업물인지 알 수 있고, 카페24 기본 스타일과 섞여 충돌하는 사고를 줄입니다.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md), [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

### _nk/ · custom.css

- **쉬운 뜻:**
  - **`_nk/`** = 새로 만든 커스텀 파일(CSS·JS·HTML 조각)을 모아 두는 **전용 폴더**.
  - **custom.css** = 카페24 원본을 건드리지 않고 색·글꼴 등을 **덮어쓰는 별도 스타일 파일**.
- **왜 중요·주의:** 카페24 원본 파일(`/layout/basic/css/`, `/css/module/`)은 **직접 수정 금지**입니다. 스킨 업데이트 때 초기화되거나 깨질 수 있어요. 모든 커스텀은 `_nk/`나 `custom.css`에서만 하세요.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md)

### strip vs EZ overlay

- **쉬운 뜻:** Easy(EZ) 코드를 어떻게 다룰지에 대한 **두 가지 방향**입니다.
  - **strip(걷어내기)** = EZ 관련 코드를 제거해서 순수 HTML 방식으로 바꾸는 것.
  - **EZ overlay(얹기/보존)** = EZ 코드를 남겨 두고 그 위에 커스텀을 얹는 것.
- **왜 중요·주의:** EZ 코드를 **통째로 걷어내는 작업은 되돌리기 어렵고** 관리자 편집 화면을 깨뜨릴 수 있어, 사용자가 분명히 지시할 때만 합니다. 평소에는 EZ 속성을 보존(overlay)하는 쪽이 안전합니다.
- **더 볼 것:** [../../.claude/skills/cafe24/SKILL.md](../../.claude/skills/cafe24/SKILL.md), [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

## 7. 안전하게 작업하기 — 점검과 함정

### 실측

- **쉬운 뜻:** 추측하지 않고 **실제 화면에서 숫자를 직접 재 보는 것**입니다. 글꼴 크기, 여백(px), 색상 같은 값을 눈대중이 아니라 브라우저 도구로 측정합니다.
- **왜 중요·주의:** "대충 이쯤"이 아니라 **정확한 값**으로 작업해야 레퍼런스와 똑같이 나옵니다. 디자인 수정 전 실측은 필수 단계입니다. (슬래시 명령 `/요소측정`)
- **더 볼 것:** [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

### run_preflight / score

- **쉬운 뜻:**
  - **run_preflight(사전 점검)** = 작업 전 "준비가 됐는지" 자동으로 확인하는 절차.
  - **score(점수)** = 결과물이 규칙을 잘 지켰는지 매기는 평가 점수.
- **왜 중요·주의:** 무작정 시작·완료하지 않고, 점검과 채점으로 실수를 미리 거르는 안전장치입니다. 점검이 통과하지 못하면 그 항목부터 해결하세요.
- **더 볼 것:** [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)

### 함정 (F코드)

- **쉬운 뜻:** 카페24 작업에서 반복적으로 사람들이 빠지는 **알려진 함정**들에 번호를 붙여 정리한 목록입니다(F1, F2, … 식). 예: `<section>` 태그에 자동으로 큰 여백이 붙는 함정.
- **왜 중요·주의:** 이미 겪은 사람이 정리해 둔 함정이라, **미리 알면 같은 실수를 피할 수 있습니다.** 이상하게 화면이 깨지면 함정 목록부터 확인하세요.
- **더 볼 것:** [../02_막혔을때/함정-INDEX.md](../02_막혔을때/함정-INDEX.md)

### optimizer · 캐시 지연 · ?v=N

- **쉬운 뜻:**
  - **optimizer(최적화)** = 카페24가 파일을 빠르게 보여주려고 미리 묶어 두는 기능.
  - **캐시 지연** = 그 때문에 파일을 올려도 화면에 **2~5분 늦게** 반영되는 현상.
  - **?v=N** = 페이지 주소 끝에 `?v=2`처럼 숫자를 붙여 **새 버전을 강제로 불러오는** 방법.
- **왜 중요·주의:** 올린 게 바로 안 보인다고 "실패했다"고 단정하지 마세요. 잠시 기다렸다가 `?v=N`으로 다시 확인하세요. 단, `?v=N`은 **페이지 주소에만** 쓰고, `<!--@css-->` 지시어에는 절대 붙이지 마세요(파일이 빠집니다).
- **더 볼 것:** [../brain/docs/CAFE24-SMARTDESIGN-AGENT.md](../brain/docs/CAFE24-SMARTDESIGN-AGENT.md)
