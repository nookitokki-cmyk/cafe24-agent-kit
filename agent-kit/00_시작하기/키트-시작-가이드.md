# 키트 시작 가이드

> **역할:** 키트를 **처음 열었을 때의 상세 walkthrough** — 표준 워크플로·필수 규칙·문서 맵을 한 페이지에 풉니다.  
> 전체 진입점은 [`../README.md`](../README.md), 온보딩 00~05 순서표는 [`README.md`](./README.md) 입니다.  
> **대상:** 카페24로 쇼핑몰·웹 디자인 작업하는 **작업자** (프리랜서·실무자·강의 중이든 동일)

이 폴더는 카페24 스킨을 AI 에이전트와 함께 빠르게 맞출 때 쓰는 **작업 키트**입니다. 레퍼런스 몰·시안을 실제 쇼핑몰에 옮기는 **작업자**를 위한 것입니다.

막히면 F코드·Q&A·프롬프트로 바로 점프하세요.

## 3. 필수 규칙 (반드시 기억)

### 3-1. 검증 점수 — 100점만 완료

모든 score 스크립트는 **`total_score = 100` only**. 100 미만 = 미완료. 「거의 맞음」은 PASS가 아닙니다.

- 상세: [`01_작업하기/workflows/06-verify-loop.md`](../01_작업하기/workflows/06-verify-loop.md) · **F33**

### 3-2. 모바일 — 관리자 OFF 필수

반응형 단일 base 스킨 작업 시 **모바일 전용 디자인 사용설정 = 사용안함** 이 전제입니다. MCP·FTP로 바꿀 수 없습니다.

> 쇼핑몰 설정 → 사이트 설정 → 쇼핑몰 환경 설정 → **모바일** 탭 →  
> **모바일 전용 디자인 사용설정** → **사용안함** → 저장

검증: 페이지 소스 `CAFE24.MOBILE_WEB=false` · **F28 · F34**

- 상세: [`brain/rules/responsive-mobile.md`](../brain/rules/responsive-mobile.md) · [`brain/rules/cafe24-admin-verify.md`](../brain/rules/cafe24-admin-verify.md)

### 3-3. F27~F34 — 레퍼런스 리뉴얼에서 자주 터지는 함정

| F코드 | 증상 | 대응 문서 |
|-------|------|-----------|
| **F27** | MO 390px에서 히어로 좌우 흰 여백 (`#contents` 92%) | [`brain/rules/ez-contents-width.md`](../brain/rules/ez-contents-width.md) · [`brain/docs/snippets/ez-contents-override.css`](../brain/docs/snippets/ez-contents-override.css) |
| **F28** | `/m/` 에 EZ 기본 스킨만 보임 | mobile OFF → 메인 URL + 390 viewport 검증 |
| **F29** | PLP·PDP가 1200px로 좁음 | `plp-full` / `pdp-full` 타입표 · [`common-pitfalls` §narrow](../02_막혔을때/common-pitfalls.md) |
| **F30** | PLP menupackage·sortby·배너 구조 불일치 | HTML 격차 먼저 · score-plp |
| **F31** | MO 장바구니 금액 줄바꿈 | PDP/장바구니 CSS · 프롬프트 §03 |
| **F32** | 상품카드 `.description` 행간 과다 | [`04-measure-first.md`](../01_작업하기/workflows/04-measure-first.md) · `sub-product.css` |
| **F33** | 「완료」인데 score 100 아님 | verify-loop 재실행 · FAIL JSON 항목만 수정 |
| **F34** | 관리자 mobile ON 상태 | §3-2 경로 안내 후 재검증 |

전체 표: [`02_막혔을때/F-상황-인덱스.md`](../02_막혔을때/F-상황-인덱스.md)

### 3-4. EZ `#contents` 92% (F27 핵심)

| 함정 | 증상 | 키트 대응 |
|------|------|-----------|
| `#contents` 92% | MO 390px 좌우 흰 여백 | `brain/rules/ez-contents-width.md` + snippet |
| paginate font-size | 게시판·목록 페이지네이션 깨짐 | `sub-paginate.css` + score-paginate |
| mobile 별도 스킨 | base만 맞고 `/m/`은 EZ 기본 | **관리자 mobile OFF** |
| line-height·spacing | PLP 카드 `.description` 느슨함 | `sub-product.css` + score-plp |

**메시지:** 「CSS 한 줄」이 아니라 **페이지 타입 + 레이어 + 실측 루프**가 필요합니다.

## 7. 작업 중 꼭 기억할 지침

1. **실측 먼저** — DevTools·Playwright 숫자 없이 「예쁘게」 수정 금지.
2. **100점** — score 스크립트 PASS는 100만. FAIL JSON → 해당 CSS/HTML만 고친 뒤 재실측.
3. **레퍼런스인입** — URL·시안이 있으면 코드 전에 타입표·격차표부터.
4. **함정** — F27 `#contents` 92%, F28 mobile 별도 스킨, F29 narrow 오적용, F3 FTP 캐시 2~5분.

예시 데모 몰 (PLP 실측·score **연습용 — 선택**):

→ [`예시-데모몰.md`](./예시-데모몰.md) — Ref/Tgt URL·스크립트. 본인 몰이 있으면 그걸로 `/레퍼런스인입` 하면 됩니다.
