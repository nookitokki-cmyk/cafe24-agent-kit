# 04. 실측 우선 (Measure First)

> **원칙:** 디자인 지시가 와도 **먼저 숫자를 모은다.** 「예쁘게」만으로는 작업하지 않는다.  
> 명령: `/요소측정` · 완료 후 `/디자인수정`

---

## 언제 쓰나

- 「헤더 바꿔줘」「여백 줄여줘」「폰트 고급스럽게」처럼 **말만 있는 요청**
- 레퍼런스 URL·Figma·스크린샷이 있을 때

---

## 흐름

```
사용자 지시
  → /요소측정 (질문 핑퐁)
  → 실측 시트 작성
  → 사용자 「예」
  → /디자인수정 또는 workflow 01/02
  → 업로드·?v=N·PC+모바일 스크린샷
```

| 항목 | 값 |
|------|-----|
| 페이지 / URL | |

| 제목 font-family / size / weight | |

> **실측 시트 = 1단계(숫자 수집).** 이 숫자를 아래 "디자인 시스템 시트"로 정리·승인해야 코드 착수.

## 디자인 시스템 시트 (실측 → 시스템 · 코드 전 게이트)

실측 숫자를 흩어진 값으로 두지 말고 **타이포 스케일(먼저) → 컴포넌트·에셋·유닛 인벤토리** 순으로 확정한다. (두뇌 문서 `brain/docs/CAFE24-SMARTDESIGN-AGENT.md` STEP 1)

### ① 타이포그래피 스케일 (먼저 확정)

| 레벨 | font-family | size (clamp) | weight | line-height | letter-spacing | 토큰 |
|------|-------------|--------------|--------|-------------|----------------|------|
| h1 (대제목) | | | | | | `--nk-fs-h1` |
| h2 (섹션제목) | | | | | | `--nk-fs-h2` |
| h3 (서브라벨) | | | | | | `--nk-fs-h3` |
| lead (리드) | | | | | | `--nk-fs-lead` |
| body (본문) | | | | | | `--nk-fs-body` |
| sm (캡션/가격) | | | | | | `--nk-fs-sm` |

폰트·팔레트는 **큰 결정 → 매번 확인**(추측 금지).

### ② 컴포넌트 · 에셋 · 유닛 인벤토리

재사용 부품을 **XANS 앵커와 함께** 확정. 코드는 이 표의 XANS 앵커에 스타일을 얹는 것으로 시작(두뇌 §5 B8-2).

| 컴포넌트 | XANS 앵커 셀렉터 (module→`.xans-*`) | 유닛/에셋 (색·radius·control-h·Phosphor) | PC | MO |
|----------|-------------------------------------|------------------------------------------|----|----|
| 버튼(1차/라인) | (module 없으면 `.nk-btn` 등 커스텀) | | | |
| 폼컨트롤(input/select/checkbox) | `.xans-*` 폼 컨테이너 | | | |
| 탭 | 예: `.xans-order-tabinfo` / `.xans-myshop-main` | | | |
| 페이지네이션 | `.xans-*` + `.ec-base-paginate`(말단) | | | |
| 표 | `.xans-board-listpackage-N` 등 | | | |
| 카드/상품 | `.xans-product-listnormal` 등 | | | |
| 아이콘 | (Phosphor 글리프 코드) | | | |

> **XANS 앵커 찾는 법**: 대상 페이지를 열어 개발자도구에서 부품의 실제 렌더 클래스(`xans-...`)를 확인. `module=`이 스트립돼도 `.xans-*`는 남는다. `ec-base-*`는 공유 클래스라 앵커로 쓰면 번짐(두뇌 §5 B8-2).

## 파일 경로 (계정 유형별 — 실측 2026-06-19)

| 계정 | 편집 루트 예 | MCP/SFTP |
|------|-------------|----------|
| **일반 몰** (디자인 SFTP) | `/{skin_code}/…` (예: `/skin14/layout/…`) | `cafe24_sftp_*` |
| **파트너 샘플몰** (웹 FTP) | **`/sde_design/base/…`**, `/sde_design/mobile/…` | FTP 클라이언트. 루트에 `/{skin_code}` **없을 수 있음** |

파트너 몰은 API `list_themes` 없이 FTP list로 **실제 폴더**를 먼저 확인.

---

## MCP로 현재값 읽기 (일반 몰)

```text
cafe24_sftp_read path=/{skin_code}/_nk/css/common.css
```

또는 HTML에서 인라인이 아닌 **linked CSS** 확인.

## 완료 조건

**디자인 시스템 시트**(타이포 스케일 + 컴포넌트/에셋/유닛 인벤토리)가 **사용자 승인**된 뒤에만 CSS/HTML 패치 시작. 실측 숫자만으로는 착수 금지.
