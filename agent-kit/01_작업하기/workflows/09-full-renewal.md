## 절차

### 0. 온보딩·환경
- `/새클라이언트` → `/접속세팅`·`/API발급`·`/MCP연결` (없으면). **작업은 연습 스킨(복제)에서** — 대표 스킨 직접 금지.

### 1. 입력·유형판별 (SSOT) — `/레퍼런스인입`
- 디자인 입력: **① 가져오기**(Figma 시안·레퍼런스 URL) **또는 ② 프리셋**(`design-tokens/presets/`에서 고르기).
- 산출: **페이지 인벤토리 + 타입표**. 이 타입표가 **유일한 유형 기준(SSOT)** — `brand-profile.json`·`skin-method-detect`의 타입은 참조만.

### 2. 토대 정리 (1회) — `/토대정리`
- base 전수 스캔 → `BASE-CSS-MAP.md`, `nk-cafe24-reset.css`, 디자인 토큰(`:root`), 컴포넌트 갤러리.

### 3. ★ 페이지 유형별 재구축 레시피 [신규 A]
> 1번 타입표의 각 페이지를 아래 트랙으로 배분. 오늘 확정한 3트랙·안전등급을 그대로 적용.

| 페이지 유형(타입표) | 트랙 | 보존 대상 | 근거·도구 |
|---|---|---|---|
| `hero-main`(메인) | 🟢 재마크업(완전) | `Layout_*` 래퍼 | `module-remarkup` |
| `plp-full`(상품목록) | 🟢 재마크업(완전) | `product_list*` 래퍼 + `anchorBoxId` 2개+ + 정의된 변수 | `module-remarkup §5` [검증됨] |
| `pdp-full`(상품상세) | 🟢 재마크업 + 버튼 이식 | `product_action` `onclick="{$action_*}"`·옵션 모듈 | `module-remarkup §5` [검증됨: 구매버튼] |
| `narrow`/정적 | 🟢 재마크업 | — | `module-remarkup` |
| `board`(게시판) | 🟡 재스타일(표 보존 권장) | 반복단위·페이징 변수 | `module-safety` 🟡 [검증필요] |
| `cart`·`order`(장바구니·주문) | 🔴 재스타일 **또는** JS 재구축 | `<table>`·`rowspan`·`{$form.*}`·`name="{$chk_name}"`·`{$action_*}` 골격 | 간단=CSS 보존 / 완전커스텀=`cart-js-rebuild.md` [검증됨] |
| `member`(회원폼) | 🔴 재스타일(보수적) | `input name`·`{$action_*}` | 🔴 [검증필요] |

- 규칙: **표시 = 반복단위만 지키면 완전 재작성 / 거래 표 = 골격 보존(de-table 금지, 완전커스텀은 JS 재구축).** (`module-remarkup §5`)
- 재마크업 후 **`check-binding.mjs` 정적검사 + 라이브 `xans-record-` 육안** 2단 검증.

### 4. ★ 전-페이지 배치 실행 [신규 B]
- 타입표 페이지를 **유형별로 그루핑** → 그룹마다 `03-reference-renewal` 3~6단계(백업→구현→업로드→격차→일괄수정)를 **1사이클씩** 순회.
- 진행상태는 `clients/{몰}/.workflow.md`에 페이지별 체크로 영속(중단·재개 가능).
- 위험 순간(거래 페이지 반영·결제 영역)엔 **사용자 「예」 게이트**.

### 5. 검증 (역할 분담 — 3중 채점 혼선 방지)
- **페이지별**: `06-verify-loop` (Phase 1~6, PASS≥90)
- **최종 통합**: `qa-loop` (3축: visual≥0.85 + override + rule) — 전 페이지 완료 후 한 번
- (단발 확인만 필요하면 `/검증`)

### 6. 라이브 반영 (초보 안전)
- **연습 스킨에서 전부 검증 통과** → 사용자 확인 → **대표 스킨 반영** → 캐시 구분(`?v=N`, 2~5분) 후 재확인.

### 7. 출력·기록
- 페이지별 변경 파일 / 백업 / Before·After(PC+모바일) / 유형별 트랙 / 남은 이슈
- `clients/{몰}/05_work-log.md` 누적. 페이지 타입별 자동 채점: `06-verify-loop`.

## 안전 (배포용 = 초보 수강생 보호)
1. **항상 연습 스킨(복제)에서.** 대표 스킨 직접 편집 금지 — 검증 통과해야 반영.
2. **거래 페이지(카트·주문·회원폼)는 표 골격 안 뜯기.** 완전 커스텀은 JS 재구축(별도 실증).
3. **위험 순간엔 확인 게이트.** AI가 어려운 건 다 하되, 라이브 반영·결제 영역은 사용자 「예」.
4. **지어낸 지식 금지** — [검증됨]/[검증필요] 라벨 준수(`evidence-labels.md`).
