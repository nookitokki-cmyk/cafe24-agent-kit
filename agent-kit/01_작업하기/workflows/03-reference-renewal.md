# 워크플로우 03 — 시안/레퍼런스 1:1 구현 (reference-renewal)

> 대상: 피그마 시안 또는 "이 사이트처럼요" 레퍼런스를 그대로 구현. (`05-reference-intake` 승인 후 진행)
> 핵심: **구현 70%는 쉽다. 나머지 30%(격차)를 토큰 아끼며 잡는 게 이 워크플로우의 존재 이유.**

## 절차 (8단계)

### 1. 시안 소스 판별
- **피그마가 있으면**: Figma MCP로 측정값(높이·색·폰트·간격) 추출
- **레퍼런스 몰/사이트면**: DevTools 실측 — 사용자가 모르면 사용법부터 안내:
  - F12 → 대상 우클릭 "검사" → 콘솔에서 `getComputedStyle($0)`
  - 잴 것: **치수·색(헥스)·타이포·동작**만. 배치 구조는 스크린샷으로 충분
- **시작 전 필수**: `/레퍼런스인입` — 페이지 인벤토리·타입표·실측시트·사용자 「예」 (`01_작업하기/workflows/05-reference-intake.md`)

### 2. 섹션 썰기 + 구현 계획표
시안을 가로로 썰어 (탑배너/헤더/히어로/진열/푸터…) 계획표 1장을 만들고 사용자 확인:

| 섹션 | 레퍼런스 실측 | 대상 파일 | 난이도 | 순서 |
|---|---|---|---|---|
| 탑배너 | 26px / #B0B5B9 / 11px / 롤링 | `_nk/inc/topbanner.html` (신규) | 하 | 1 |

- 현재 스킨 매핑은 MCP 3연타: `cafe24_list_themes` → `cafe24_sftp_list` → `cafe24_sftp_read`

### 2.5. 토대 정리 (섹션 구현 전 1회) — `/토대정리`
- **base 전수 스캔** → `BASE-CSS-MAP.md` (이 클라 고유 함정 색출)
- **`nk-cafe24-reset.css` 적용** (`@css` 맨 위 + `<body class="nk-skin">`) — base 함정 일괄 중화
- **디자인 토큰 세팅** (1단계 실측치 → `--nk-point` 등 `:root` 변수)
- 상세: `/토대정리` 스킬. 이게 돼 있어야 6단계 "그 줄만 수정"이 성립한다.

### 3. 섹션 구현 — 1사이클씩
- **한 사이클 = 한 섹션**: 백업 → 구현(신규 조각 + @import) → 업로드(컨펌) → 라이브 확인
- 지시문 3요소를 스스로 지킨다: ①범위 한정 ②실측치 동봉 ③완료 기준(라이브 스크린샷)
- 상세 생성 규칙은 `02-skin-build-standard.md` 3~4단계와 동일

### 4. 격차 수집 (수정 금지 구간)
- 모든 섹션(또는 한 섹션) 구현 후, 레퍼런스 vs 내 구현을 **PC+모바일 스크린샷으로 나란히** 비교
- 다른 점을 **전부** 격차표로 수집 — **수집이 끝나기 전에는 한 글자도 고치지 않는다**

### 5. 격차표 작성 (형용사 금지)
양쪽 다 `getComputedStyle`로 실측해 숫자로 채운다:

| # | 속성 | 레퍼런스 실측 | 내 구현 | 지시 |
|---|---|---|---|---|
| 1 | 높이 | 26px | 60px | 26px로 |
| 2 | 배경 | #B0B5B9 | #999999 | #B0B5B9로 |

### 6. 일괄 수정 — 왕복 1회
- 격차표 **통째로 1회 지시**: "표대로 일괄 수정. 다른 부분은 건드리지 마."
- **파일 재생성 금지 — 해당 값(토큰 줄)만 수정.** 토큰 설계가 돼 있으면 보통 `:root` 몇 줄로 끝
- 재업로드(자동 백업) → 라이브 재실측 → **숫자로 PASS/FAIL 판정**

### 7. 캐시 구분 (헛수정 방지)
- 수정했는데 그대로면: `?v=N` → 하드 리로드 → 2~5분 대기 **후에** 격차 재판정
- 캐시를 격차로 오판해 맞는 값을 또 바꾸는 것이 최악의 토큰 누수

### 8. 출력 + 기록
- 섹션별 변경 파일 / 백업 / 라이브 URL / Before·After 스크린샷 / 격차표(PASS 표시) / 남은 이슈
- `clients/{몰}/05_work-log.md`에 왕복 횟수와 교훈 기록
- 페이지 타입별 자동 채점: `06-verify-loop.md` (Phase 1~6, PASS≥90)

## 토큰 절약 5원칙 (요약)

1. **격차표부터** — 수집 끝까지 수정 금지
2. **형용사 대신 실측치** — "60px을 26px로"
3. **한 번에 묶어 1회 지시** — 왕복 N회 → 1회
4. **재생성 금지, 그 줄만**
5. **격차인지 캐시인지 먼저 구분**
