사용자가 `/키트시작` 을 요청했습니다. **한 번에 질문 하나.** 답 없으면 다음 단계 금지.

## Q1. 워크스페이스

「`cafe24-agent-kit` 폴더를 Cursor/Claude Code **루트**로 열었나요? (zip 해제 후 최상위 폴더)」

- 아니오 → zip 해제 · 해당 폴더를 워크스페이스로 연 뒤 다시 `/키트시작`

## Q2. Python 의존성

「터미널에서 `cd mcp && pip install -r requirements.txt` 를 실행했나요? (에러 없이 끝났나요?)」

- 실패 → Python 3.10+ 확인 · `py -3` / `python3` 시도

## Q3. import 검증

「`cd mcp && python -c "import server"` — 에러 없이 끝났나요?」

- `No module named auth` → 배포본 **v2.0.1+** 인지 확인 (`mcp/auth/`, `mcp/backends/` 폴더)

## Q4. MCP 등록 (아직이면)

「`.cursor/mcp.json` (Cursor) 또는 Claude Code MCP 설정을 했나요?」

- 아니오 → **`/MCP연결`** 진행

## Q5. smoke 기대치

「`cd mcp && SMOKE_PREFLIGHT_ALL=0 python smoke_test.py` 를 돌렸나요?」

| 결과 | 의미 |
|------|------|
| **5/9 통과** (config·토큰 없음) | **정상** — OAuth 전 단계 |
| **9/9+** (토큰·SFTP 있음) | 완전 연결 |
| tools/list ERROR | `/MCP연결` 재확인 |

> config 없을 때 exit 0 이 나오면 정상 (v2.2+). ExceptionGroup 이면 키트 버전 확인.

## 완료 시 출력

```
[키트시작 완료]
- import server: OK
- smoke: {N}/9 (config 없으면 5/9 정상)
- 다음: **/API발급** (카페24 토큰 — 첫 설치면 보통 여기로) → 그다음 /새클라이언트 (새 몰) 또는 /접속세팅
```

## 금지

- 이 명령에서 SFTP 업로드·HTML 수정 금지
- `client_secret` 채팅 입력 금지
