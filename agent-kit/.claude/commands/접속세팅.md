사용자가 `/접속세팅` 을 요청했습니다. **한 번에 질문 하나.** 답 없으면 접속·업로드 금지.

> ⚠️ **파트너센터 = 웹 FTP.** SFTP를 묻지 말 것. 접속정보는 `sftp_{몰ID}.json`(FTP·SFTP 공용)에 저장하되, 파트너센터는 그 안에 **`"protocol": "ftp"` + `"port": 21`** 필수.

## 질문 순서 (대본)

### Q1. 몰 종류
「**파트너센터 샘플몰**인가요, **일반 운영 몰**인가요?」

| 답 | 안내 |
|----|------|
| 파트너 | 웹 FTP 경로 → `00_시작하기/03` §파트너. ⚠️ 호스트는 발급/화면 값. |
| 일반 | 디자인 SFTP 권한 신청 → `00_시작하기/03` §일반. |

### Q2. 몰 ID
「쇼핑몰 주소를 알려주세요. (`https://____.cafe24.com`)」

→ 앞부분을 몰 ID로 기록. `02-몰ID-찾기.md` 참고.

### Q3. 접속 정보 (해당되는 쪽만)
- 파트너: 「웹 FTP **호스트·포트(보통 21)·아이디**를 화면에 나온 그대로 알려주세요 (비밀번호는 채팅에 X).」 → `sftp_{몰ID}.json` 에 **`"protocol":"ftp"`** 로 저장
- 일반: 「디자인 SFTP **호스트·포트**를 발급 메일/관리자에 나온 그대로 (비밀번호 X).」 → `sftp_{몰ID}.json` (protocol 생략 = SFTP)

> 설정 파일 형식 예시: `00_시작하기/03-접속-웹FTP-vs-SFTP.md` §MCP 설정

**금지:** `{몰ID}.ftp.cafe24.com` 이나 `22` 포트를 **기본값으로 가정** ❌ AUDIT §A-2e,f

### Q4. MCP / 스킨 목록
「MCP `cafe24_list_themes` 를 쓸 수 있나요? (예/아니오)」

- 예 → 결과를 표로 정리:

| skin_no | skin_code | editor_type | skin_name |
|---------|-----------|-------------|-----------|

`editor_type` 이 **H**가 아니면 HTML/SFTP 자동 작업 중단 → Easy 안내 §D.

- 아니오 → 관리자 디자인보관함에서 **디자인 번호·코드** 스크린샷 요청.

### Q5. 작업 스킨 확정
「위 표에서 **어느 skin_code** 를 수정할까요?」

→ 업로드 경로(몰 종류별 다름):
- **파트너센터(FTP):** `/sde_design/base/` · `/sde_design/mobile/` (⚠️ `/{skin_code}` 아님)
- **일반 운영몰(SFTP):** `/{skin_code}` ⚠️ 실측 (`skin_no` 와 폴더 숫자 불일치 경고 §E-2b)

### Q6. 백업
「파일 올리기 전 **백업** OK인가요?」

OK 받기 전 `cafe24_sftp_upload` 금지.

## 완료 시 출력

```
[접속세팅 완료]
- 몰 종류: ...
- 몰 ID: ...
- 접속: (호스트:포트 — 비밀번호 제외)
- skin_no / skin_code / SFTP 경로
- editor_type
- 다음: /API발급 (미연결 시) 또는 /요소측정
```
