## 2. 이렇게 만들어졌다 (3단계)

1. **제작·페이지별 QA** — 데모몰 402307에서 스킨을 페이지군 단위(wave)로 제작. 각 wave마다 rule(문법·함정) / override(base 충돌) / visual(레퍼런스 대조) 3축 게이트 + 라이브 인터랙션 e2e를 통과시킴.
2. **브랜드 중립화** — 브랜드 고유 요소를 전수 실측(16항목)해 중립값으로 매핑(§4). 핵심: 브랜드가 **토큰 값·콘텐츠·이미지 계층에만** 존재하고 구조(셀렉터·DOM·module 배선·JS)에는 없음을 실측으로 입증 → 값 교체만으로 중립화 완료. 이어 파일 헤더·주석의 내부 경로 흔적까지 제거(새니타이즈)한 31파일을 빌드.
3. **재현성 재검증** — 빌드된 31파일을 **다른** 데모몰 399293에 그대로 설치하고 최종 QA(§3)를 전부 통과. "한 몰에서만 우연히 되는 스킨"이 아님을 증명.

## 4. 중립화 매핑 요약

> 브랜드 원값(색상 hex·브랜드명·원본 이미지)은 공개 문서에 싣지 않습니다. 아래는 **계층별 치환 방식** 요약.

| 계층 | 브랜드 요소 | 중립 치환 | 방식 |
|---|---|---|---|
| 토큰 값 (5줄) | 포인트색 · hover색 · 공지바색 · 디스플레이 폰트 · 본문 폰트 상속 | `#222222` · `#000000` · `#b0b5b9` · Pretendard · Pretendard(상속 절단) | `nk-tokens.css` 값 줄만 교체 |
| 콘텐츠 (8건) | 헤더·푸터 워드마크 / 드로어 브랜드 / 히어로 카피 / 공지 문구 / 세리프 폰트 CDN `<link>` / 주석 | module 안은 `{$mall_name}` 변수, module 밖은 정적 플레이스홀더(`SHOP`·`YOUR BRAND`) + `<!-- 설치: ... -->` 마커, 폰트 링크는 제거 후 옵트인 안내 주석 | 텍스트 노드·head 1줄 치환 |
| 이미지 (4매) | 히어로 1 + 배너 3 | CSS **그라디언트 플레이스홀더** (이미지 파일 미동봉 — 라이선스·용량·브랜드 리스크 0) | `url()` → `linear-gradient()` |
| 주석 헤더 | 파일 헤더의 브랜드·내부 경로 표기 | 역할·로드 순서·스코프 규칙만 남긴 중립 헤더로 재작성 | 주석만 수정 |

**구조 불변 증명**: 셀렉터·DOM 구조·module 배선·JS를 수정해야 바뀌는 항목 = **0** (전수 매핑 16항목 중 값 5 + 콘텐츠 8 + 이미지 2군 + 변경 불요 1). 중립화 전후 비주석 코드 라인 변경 0 — 402307에서 검증된 코드가 **그대로** templates에 들어 있습니다.
