# 검증 템플릿 (_verified-template) — 비코더용 설치 안내

> 실제 카페24 데모몰 2곳에서 QA 게이트를 전부 통과한 **브랜드 중립 스킨 템플릿**입니다.
> 새 몰을 시작할 때 이 템플릿을 올리면, 빈 스킨이 아니라 **검증된 토대**(디자인 토큰 + base 함정 상쇄 + 공통 컴포넌트 + 핵심/확장 페이지 20종) 위에서 시작합니다.
> 어떻게 만들어지고 검증됐는지: [`CASE-STUDY.md`](CASE-STUDY.md)

---

## 1. 이게 뭔가요?

- 카페24 **스마트디자인(HTML) 스킨**용 템플릿 — 소스 **45파일** (`src/` 폴더: HTML 27개, CSS 16개, JS 2개).
- 커버 페이지: **메인 / 상품목록(PLP) / 상품검색 / 상품상세(PDP) / 로그인 / 회원가입 / 가입완료 / 아이디·비밀번호 찾기 / 장바구니 / 마이쇼핑 / 관심상품 / 마이쿠폰 / 쿠폰존 / 회사소개 / 이용안내 / 게시판 목록·읽기·쓰기 / 상품게시판 목록** + 공용 레이아웃·팝업 레이아웃.
- 나머지 페이지(주문·결제·PG 등 C그룹)는 몰 **기본(스톡) 페이지를 그대로** 씁니다 — 공용 레이아웃(헤더·푸터·폰트)에 더해, `nk-stock.css` **스톡 톤 레이어**가 표·버튼·안내문·탭·페이지 타이틀 같은 카페24 공통 부품(ec-base-*)을 토큰으로 리스타일해서 스톡 페이지도 템플릿과 같은 톤으로 보입니다. (HTML 무변경 — 주문·결제 흐름 안전)
- 모든 색·폰트·치수는 `src/_nk/css/nk-tokens.css` **한 파일의 변수(토큰)** 로 관리됩니다. 다른 CSS는 전부 `var(--nk-*)`로만 참조 — 색을 바꿀 때 여기저기 고칠 필요가 없습니다.

### 폴더 구조 (`src/` 내용물 = 스킨 루트에 그대로)

```
src/
├── index.html                  ← 메인 라우팅 스텁
├── layout/basic/               ← 공용 레이아웃 · 메인 본체 · 팝업 레이아웃
├── product/                    ← 상품목록(list) · 상품검색(search) · 상품상세(detail)
├── member/                     ← 로그인(login) · 회원가입(join) · 가입완료 · 아이디/비밀번호 찾기
├── order/                      ← 장바구니(basket)
├── myshop/                     ← 마이쇼핑(index) · 관심상품(wish_list) · 마이쿠폰(coupon)
├── coupon/                     ← 쿠폰존(coupon_zone)
├── shopinfo/                   ← 회사소개(company) · 이용안내(guide)
├── board/free/                 ← 게시판 목록(list) · 읽기(read) · 쓰기(write)
├── board/product/              ← 상품게시판 목록(list)
└── _nk/
    ├── css/  (16개)            ← nk-tokens(토큰 정본) → nk-cafe24-reset → nk-base → nk-components → nk-stock(스톡 톤) + 페이지별
    ├── inc/  (4개)             ← 헤더 · 푸터 · 퀵메뉴 · 오버레이 조각
    └── js/   (2개)             ← PDP 탭 · 이용안내 탭 스크립트
```

## 2. 설치 순서

1. **연습·새 몰용 스킨 사본을 만듭니다.** 카페24 관리자 → 디자인(PC) → 스킨 **복사** 또는 새 스킨 추가. **운영 중인 대표 스킨에 바로 올리지 마세요.**
2. **FTP 접속 정보를 준비합니다.** 처음이면 채팅에 `/접속세팅` — 몰 ID·SFTP·스킨 번호를 한 번에 세팅해 줍니다.
3. **`src/` 안의 내용물을 통째로 스킨 폴더에 업로드합니다.** 폴더 구조를 바꾸지 말고 그대로(예: `src/_nk/` → 스킨의 `/_nk/`, `src/layout/basic/layout.html` → 스킨의 `/layout/basic/layout.html`). 같은 이름 파일은 덮어씁니다.
4. **미리보기로 확인합니다.** 관리자 스킨 목록에서 미리보기 — 반영이 안 보이면 캐시일 수 있습니다(2~5분, F3). `/캐시확인` 참고.
5. **내 브랜드를 입힙니다.** 아래 3장 — 토큰 값 몇 줄이면 됩니다.
6. 확인이 끝난 뒤에만 **대표 스킨으로 지정**합니다.

> **관리자 모바일 설정은 OFF** — 이 스킨은 PC/모바일을 **단일 스킨 반응형**으로 처리합니다. 관리자 모바일 웹이 켜져 있으면 모바일에서 다른 스킨이 뜹니다(F34). `brain/rules/responsive-mobile.md` 참고.

## 3. 내 브랜드로 바꾸는 법 (토큰 값)

### 3-1. 색 — `src/_nk/css/nk-tokens.css` 값만 수정

| 토큰 | 현재(중립 기본값) | 바꾸면 |
|---|---|---|
| `--nk-point` | `#222222` | 포인트색 — 버튼·CTA·포커스·액센트 전부 한 번에 |
| `--nk-point-active` | `#000000` | 포인트 hover/pressed (포인트색보다 한 단계 어둡게) |
| `--nk-topbar-bg` | `#b0b5b9` | 상단 공지바 배경 |
| `--nk-bg` / `--nk-bg2` | `#faf9f5` / `#f5f0e8` | 페이지 배경 / 연한 서피스 |

값은 **여기서만** 바꾸세요. 다른 CSS 파일의 색을 직접 고치면 토큰 규율이 깨집니다.

### 3-2. 브랜드명 — 자동 1곳 + 교체 마커 3곳

- 헤더·푸터 로고는 `{$mall_name}` **module 변수** — 관리자에 등록된 쇼핑몰 이름이 **자동으로** 나옵니다. 손댈 필요 없음.
- 아래 3곳은 `<!-- 설치: ... -->` 주석이 붙어 있으니 검색해서 교체하세요:
  - `_nk/inc/nk-header.html` — 모바일 드로어의 `SHOP`
  - `layout/basic/main.html` — 히어로의 `YOUR BRAND` / `Your tagline goes here.`

### 3-3. 폰트 — 기본 Pretendard, 세리프 디스플레이는 옵트인 (2줄)

기본은 kit 표준 **Pretendard**입니다. 제목만 세리프 폰트로 바꾸고 싶으면:

1. `layout/basic/layout.html`과 `layout/basic/main.html`의 head(옵트인 안내 주석 자리)에 구글 폰트 `<link>` 1줄 추가
2. `nk-tokens.css`의 `--nk-font-display` 값 **맨 앞에** 폰트명 추가 (예: `"MySerif", "Pretendard", ...`)

`--nk-font-body`는 건드리지 마세요 — 본문은 항상 Pretendard 유지가 kit 표준입니다.

### 3-4. 이미지 — 그라디언트 플레이스홀더 → 내 이미지

히어로·배너는 이미지 파일 대신 **CSS 그라디언트 플레이스홀더**로 들어 있습니다(이미지 없이도 렌더 정상). `src/_nk/css/nk-main.css`에서 `중립 플레이스홀더` 주석을 검색해 `background-image`를 `url("/_nk/img/내이미지.jpg")` 형태로 교체하고, 이미지를 FTP로 `/_nk/img/`에 올리세요.

## 4. 주의사항

- **[필수] kit-update 시 이 폴더는 통째로 갱신(덮어쓰기)됩니다.** 실제 몰 작업은 반드시 이 폴더를 `clients/{몰ID}/`로 **복사한 뒤** 그 사본에서 하세요(`/새클라이언트` 참고). 이 폴더 안에서 직접 작업하면 다음 업데이트 때 수정분이 사라집니다.
- **주문서(`order/orderform.html`)는 손대지 않습니다.** 템플릿에 주문서 HTML이 없는 것은 누락이 아니라 **"몰 기본(스톡) 주문서를 그대로 쓴다"가 검증된 정답**이기 때문입니다(결제 연동 리스크). `_nk/css/nk-orderform.css`는 [미사용·보존] 참고본 — 로드되지 않으며 지우지 않아도 됩니다.
- **`module="..."` 속성과 `{$...}` 변수를 지우지 마세요.** 상품·장바구니 수·로그인 상태 등 실데이터는 module이 붙은 태그 안에서만 채워집니다. 반복 상품 블록(`anchorBoxId_{$product_no}`)도 구조를 유지하세요.
- **백업과 롤백(되돌리기):** 비코더/비개발자 기준으로 가장 안전한 방법은 업로드 전 원격 스킨을 백업하고, 문제가 생기면 백업본으로 원복하는 것입니다. SFTP 업로드 전에는 `/_backup/{날짜}/` 같은 백업 폴더를 만들고, 업로드 후 화면이 깨지면 새로 올린 파일을 지우거나 백업 파일을 같은 위치에 다시 올려 복구하세요.
- **캐시(F3):** FTP 업로드 후 2~5분 내 반영. 안 바뀌면 `/캐시확인`.
